- Ornstein-Uhlenbeck (mean-reverting SDE) — canonical for market dynamics
- Hawkes process (self-exciting point process) — huge for market microstructure
- Lévy processes (α-stable, VG, NIG)
- SDE solvers: Euler-Maruyama, Milstein, Runge-Kutta for SDEs
- Brownian bridge, geometric Brownian motion
- Fokker-Planck equation
- Cox process (doubly stochastic Poisson)

**2. State Space Models & Filtering**
- Linear Kalman filter (discrete + continuous)
- Extended Kalman filter (EKF)
- Unscented Kalman filter (UKF)
- Particle filter / Sequential Monte Carlo
- Kalman smoother (RTS)
- Hidden Markov Models (Viterbi, Forward-Backward, Baum-Welch) — PROMISED in mixture.rs, not there
- State-space ARIMA

**3. ARMA/ARIMA/VAR family**
- MA(q) fitting (Innovations algorithm or MLE)
- ARMA(p,q) — mixed model
- ARIMA(p,d,q) — differenced ARMA
- SARIMA — seasonal
- VAR (Vector Autoregression)
- VECM + Johansen cointegration test

**4. Mathematical Finance**
- Black-Scholes + all Greeks (delta, gamma, vega, theta, rho)

- Extreme value theory: GEV, GPD, Hill estimator, POT method
- Value at Risk + Expected Shortfall
- Yield curve fitting (Nelson-Siegel, Svensson)

**6. Physics Engines** (entirely absent)

## Test Quality Assessment

### Gold Standard: MUCH LARGER THAN INITIALLY REPORTED ← CORRECTION
- `gold_standard_parity.rs`: scipy/numpy/sklearn/faer-verified oracle comparisons with proper abs+rel tolerance
- **489 tests** covering: softmax, dot product, L2 distance, linear regression, logistic regression, DBSCAN, accumulate primitives, descriptive stats (f06), hypothesis tests (f07), nonparametric/rank tests (f08), information theory (f25), neural primitives (f23), manifold geometry (f28), KNN (f35), optimization (f05), graph algorithms (f29), SVD vs faer
- My initial audit read only the first ~100 lines and reported "only covers softmax" — completely wrong. The file is 502KB.
- Remaining genuine coverage gaps: survival (KM, Cox PH), ARMA/GARCH, KDE, physics

### Adversarial Boundary: EXCELLENT, found a real bug
- adversarial_boundary.rs through _10.rs: boundary conditions, NaN inputs, degenerate cases
- **ACTIVE BUG**: `tda_rips_h0_single_point` (adversarial_boundary.rs) documents that `rips_h0` returns empty diagram for n=1 — should return 1 pair with death=∞. Bug is in production code, test documents it.

### In-Module Tests: MIXED
- ✓ Strong: survival.rs, time_series.rs (exact values), hypothesis.rs (known outcomes)
- ⚠ Weak: bayesian.rs (MH chain mean within 0.3 — directional not exact), GARCH (range checks only)
- ⚠ Trivial: `ewma_constant_returns` only checks `len() == 20 && > 0.0` — not a math truth test
- ✗ No oracle verification for: hypothesis tests, KM, AR, GARCH, FFT, SVD

### Scale Ladder Tests: PERFORMANCE, not correctness
- `scale_ladder*.rs` files test scaling behavior, not mathematical accuracy

## Structural Observations (the good stuff)

**The accumulate+gather decomposition is the most disciplined I've seen.** Every module header annotates Kingdom A/B/C. Every algorithm maps to the primitives. This is baked in, not retrofitted.

**series_accel.rs is a hidden gem.** 2229 lines, Cohen-Villegas-Zagier alternating series, Shanks/Wynn-ε transform. Graduate-level series acceleration that standard libraries don't have.

**bigfloat.rs contains Riemann zero finding.** Hardy-Z function, numerical search for Riemann zeros. No external dependencies. Remarkable.

**fold_irreversibility.rs (3119 lines) + equipartition.rs are original research.** Longest files in the codebase. Full thermodynamic framework. Not standard fare.

**mixture.rs says HMM but has none.** This is the most surprising discrepancy: a module docstring promising 3 algorithms with only 1 delivered.

**TDA adversarial tests caught a real bug.** The process works — adversarial testing found a mathematical error in rips_h0. But the bug is still unfixed.
