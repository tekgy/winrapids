# Peak 1 logbook — campsites 1.2, 1.6–1.13

## Campsite 1.2 — Rust crate scaffold

Created `crates/tambear-tam-ir/` with `Cargo.toml` (zero deps beyond std) and `src/lib.rs` wiring five modules: `ast`, `print`, `parse`, `verify`, `interp`, plus a `#[cfg(test)] pub mod fixtures` for shared test data.

Design decision: fixtures live in their own module rather than being duplicated across tests. Every test that needs a `variance_pass_program()` or `sum_all_add_program()` imports it from `crate::fixtures`. This prevents silent drift between test programs.

## Campsite 1.6 — Text printer

`src/print.rs`: `print_program(prog: &Program) -> String`.

Key invariant: f64 constants always print as bit-exact hex (`0x<16 lowercase hex digits>` via `f64::to_bits()`). This guarantees round-trip equality for -0.0, NaN bit patterns, subnormals, infinities — no special-casing needed.

Every op maps to a single canonical text line. Loop bodies indent by 2 spaces. Functions include `entry:` label.

Tests: header format, hex encoding of 1.0 (0x3ff0000000000000), -0.0, +inf, -inf, variance_pass structural check (contains `kernel variance_pass(`, `buf<f64> %data`, `loop_grid_stride %i in [0, %n)`, `reduce_block_add.f64`).

## Campsite 1.7 — Text parser

`src/parse.rs`: `parse_program(src: &str) -> PResult<Program>`.

Recursive descent. Lexer strips `//`-comments, tokenizes line-by-line. Register parsing handles `%name` and `%name'` (prime suffix). f64 literals accept: `0x<16hex>` (bit-exact), decimal float (for hand-written files), `inf`, `-inf`, `nan`.

Three `include_str!` tests parse the hand-written `.tam` reference programs from disk. `roundtrip_variance_pass` test: build AST → print → parse → assert programs_bit_eq. This exercises the full print→parse→compare cycle for a realistic program.

Key bug fixed: `print::print_program(...)` unresolved in test — needed explicit `use crate::print::print_program;` import.

## Campsite 1.8 — Round-trip property test (10,000 programs)

`src/proptest.rs`: LCG-based random program generator + bit-equality comparator + 10,000-iteration round-trip test.

No `rand` crate dep. Used Knuth's 64-bit multiplicative LCG: `state = state * 6364136223846793005 + 1442695040888963407`. Deterministic, reproducible at seed `0xdeadbeef_cafef00d`.

Generator produces programs with:
- 0–2 funcs (each: one of fmul/fadd/fneg/fsqrt on a single arg)
- 1–3 kernels per program
- Each kernel: 1–2 input buffers, 1–4 output slots, 1–4 accumulators, 0–2 constants, one grid-stride loop accumulating via fadd/fmul/fsub

Bit-equality comparator: uses `va.to_bits() == vb.to_bits()` for `ConstF64` values (avoids NaN != NaN under PartialEq). All other ops use derived PartialEq.

Result: **10,000 programs, zero divergences**. Round-trip property holds.

`#[allow(dead_code)]` on the Rng impl — `next_i32` and `next_name` are reserved for future random integer op coverage.

## Campsite 1.9 — Verifier

`src/verify.rs`: `verify(prog: &Program) -> Vec<VerifyError>`.

Checks:
1. Undefined register references (use before define)
2. SSA double-define (same name defined twice)
3. Type mismatch: `fadd` on i32 register, `fmul` on pred, etc.
4. `ret` op in a kernel (only allowed in funcs)
5. Missing `ret` in a func
6. More than one `loop_grid_stride` per kernel (Phase 1 limit)

Five "broken program" tests, each triggering exactly the error it's named for. One clean-program test (`verify_variance_pass_clean`).

Implementation note: type inference is forward-pass only. Each op defines its dst type and checks src types against the type map. No backward inference needed since the IR is already typed at the op level.

## Campsites 1.10–1.13 — CPU interpreter

`src/interp.rs`: `Interpreter<'p>` with `run_kernel` and `call_func`.

`Val` enum: `F64(f64)`, `I32(i32)`, `Pred(bool)`. `Env` is a `HashMap<String, Val>` with a `promote_phi_outputs()` method that copies `%acc'` → `%acc` at each loop iteration end.

Buffers passed as `&[(&str, &mut Vec<f64>)]` — named-tuple form. The interpreter looks up buffer registers by name. This is more robust than positional indexing.

Transcendental stubs: `tam_exp`, `tam_ln`, `tam_sin`, `tam_cos`, `tam_pow` all `panic!` with a message noting they're wired at campsite 5.2. No `f64::exp`, `f64::sin` etc. called anywhere. Invariants I1 and I8 enforced at the code level.

FAdd/FMul comments: `// Never fma — I3`. The intent is stated at every arithmetic op.

`reduce_block_add` on CPU: `slice[slot] = val` — direct store, because on CPU the single "block" contains all elements. Matches the spec's Q4 note about CPU vs GPU asymmetry.

Tests:
- `sum_all_add_empty` — zero-length slice returns 0.0
- `sum_all_add_single_element` — [42.0] returns 42.0
- `sum_all_add_1_to_10` — [1..=10] returns 55.0
- `variance_pass_matches_expected` — [2,4,4,4,5,5,7,9] returns sum=40, sum_sq=232, count=8
- `fsqrt_correct` — sqrt(4.0) = 2.0, sqrt(2.0) ≈ 1.4142...
- `select_f64_branch_free` — select.f64 on pred true/false
- `parse_and_run_sum_all_add` — parse from include_str! then run: end-to-end test
