## Initial Notes

Current oracle runner takes a single candidate closure. Cross-backend testing (I10) needs to run the same oracle entry against CPU backend and PTX backend and diff. Design: run_oracle_pair(entry, backend_a, backend_b) -> (OracleReport, OracleReport, CrossBackendDiff). When does this become necessary? When Peak 3 PTX backend can produce f64 exp results.
