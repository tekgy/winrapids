## Initial Notes

bit_exact_checks handles +0.0 vs -0.0 via exact hex pattern (0x0000000000000000 vs 0x8000000000000000). A NamedConstraint variant 'positive_zero' would be more readable. Currently not in Phase 1 registry because hex is clear enough for 2 cases. If more zero-sign checks accumulate across oracle entries, promote to named constraint.
