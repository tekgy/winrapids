## Initial Notes

CPU interpreter does 'one block = all elements' while PTX writes partial sums needing host fold. campsite 3.12 needs to test the FOLDED value, not out[0] of raw GPU output. This is a correctness trap: a test that reads out[0] directly would pass on CPU (correct) and silently pass on GPU (only the first partial sum, wrong). Need a specific test that exercises the fold path.
