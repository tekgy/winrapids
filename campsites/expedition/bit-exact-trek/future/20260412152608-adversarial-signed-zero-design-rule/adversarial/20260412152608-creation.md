# adversarial-signed-zero-design-rule

Created: 2026-04-12T15:26:08-05:00
By: adversarial

## Initial Notes

Formalize the 'return x not return ±0_constant' rule as a named design invariant. Six of seven libm review docs had this bug. The rule: in any zero-detection branch of a libm function, never return a zero constant — always return x. Exceptions: only when the output sign is genuinely independent of input sign (exp(-0)=1.0, cosh(-0)=1.0) — those use bit_exact_checks entries, not early-return branches. Propose adding this as I12 or as a sub-constraint of I8.
