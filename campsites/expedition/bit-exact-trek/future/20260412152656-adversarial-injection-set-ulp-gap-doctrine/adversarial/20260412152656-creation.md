## Initial Notes

The injection-set ULP gap (inputs not in reference binary → runner uses candidate as own reference → ULP=0 regardless of sign) is a systemic oracle design problem, not just an exp-specific issue. Document this as a formal oracle design doctrine: for ANY function, corpus inputs that produce ±inf or ±0 outputs MUST have corresponding constraint_checks entries (infinite_positive, positive_zero, etc.) OR must be verified to be in the reference binary. Add this doctrine to oracle-runner-design.md.
