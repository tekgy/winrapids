## Initial Notes

The third candidate from aristotle's synthesis for 'what becomes a registry next' was 'ops' — the set of .tam IR primitives themselves. This one is the most abstract and probably the last to crystallize. It would take the form: an ops/ registry where each .tam op has a spec entry naming its parameters, order semantics, NaN behavior, rounding mode requirements, backend lowering obligations, and which other registries (order_strategies, oracles, device_capabilities) it cross-references. Basically the op-reference.md doc from peak1-tam-ir graduated into a named artifact with review-time enforcement. Trigger for crystallization: when pathmaker wants to add an op and the review process asks 'which of these properties has this op declared?' in a way that can't be answered informally. Probably won't happen until Peak 2 has added 10+ libm-specific ops and the Peak 1 op set is no longer one person's mental model.
