## Initial Notes

The meta-goal deconstruction (Phase 5 Move v5) stated the trek's compositional claim as three preconditions: IR-precision + faithful-lowering + IEEE-754 compliance for ops used. But the TamSession persistent store has its OWN set of preconditions for content-addressing to work correctly across sessions: (1) the hash function is deterministic over the output bytes, (2) the output bytes are themselves deterministic (inherits trek's compositional claim), (3) the cache key is stable across versions (op-name + precision + strategy + parameters all need to be part of the key and not drift). Nobody has named TamSession's preconditions as an independent chain. Worth a sub-deconstruction. Does the same three-precondition shape recur, or is this a different structure?
