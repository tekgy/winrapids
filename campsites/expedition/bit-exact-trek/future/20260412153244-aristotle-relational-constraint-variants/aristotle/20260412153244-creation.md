## Initial Notes

Scientist's NamedConstraint registry has three Phase-1 variants (nonzero_subnormal_positive, finite, infinite_positive). All three are CLASS MEMBERSHIP checks: 'is this output in class X?'. Different category possible: RELATIONAL CONSTRAINTS that test properties BETWEEN multiple outputs or between input and output. Candidates: is_monotonic_on_interval (outputs along a sorted input sweep must be non-decreasing), is_idempotent (f(f(x))==f(x) for relevant ops), is_involutive (f(f(x))==x for abs of abs, for example), satisfies_identity (sin²+cos²=1 is a two-output relational constraint). Scientist's current registry shape doesn't support relational constraints; adding them would be a structural extension. Not urgent for Phase 1 (bit_exact and class membership cover libm's needs). Becomes interesting when the method catalog grows beyond libm into multivariate tests. Flag for review when scientist's registry has 5+ entries.
