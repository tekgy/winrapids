## Initial Notes

Gap noticed while thinking about the corner-ownership table: IEEE 754 defines five exception flags (invalid, divide-by-zero, overflow, underflow, inexact) that trap or set status bits. Tambear's invariants DON'T currently say anything about these flags. Do we care? Two possible answers: (1) no, because tambear uses the returned value (inf / NaN / subnormal / exact) as the signal, and the flags are redundant. (2) yes, because some libm functions internally check flags to choose code paths, and if our lowering doesn't preserve flag semantics, a backend could produce the right value but skip a check. The question matters specifically for tambear-libm's implementation of pow(), which uses flag-checking in some references. Not urgent now. Worth a math-researcher thought when Peak 2 gets to pow(). Campsite as a placeholder for that future thought.
