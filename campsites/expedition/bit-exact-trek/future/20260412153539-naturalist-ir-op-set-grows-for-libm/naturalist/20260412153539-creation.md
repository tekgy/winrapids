# naturalist-ir-op-set-grows-for-libm

Created: 2026-04-12T15:35:39-05:00
By: naturalist

## Initial Notes

Cross-peak dependency observation that the team noticed twice (navigator's 2.1 approval flagged it, parallel-naturalist's Entry 006 review-pass also caught it): the .tam IR op set sized for USER kernels is not automatically sized for LIBM kernels. Math-researcher's exp/log designs needed ops that weren't in Phase 1: ldexp.f64, f64_to_i32_rn, f64_to_bits, bits_to_f64 (bit reinterpretation for exponent extraction). These are not exotic — every libm implementation needs them — but they're the kind of op that only becomes visible when someone writes actual libm code. Pathmaker added them. General principle worth naming: **when a new major consumer comes online, the IR needs a re-audit for the ops that consumer requires, even if the current IR passes its own tests.** Same lesson will apply when Peak 5 CPU interpreter arrives (probably no new ops), Peak 3 PTX assembler arrives (probably no new ops but new lowering requirements), and any hypothetical Peak 8 / new language surface.
