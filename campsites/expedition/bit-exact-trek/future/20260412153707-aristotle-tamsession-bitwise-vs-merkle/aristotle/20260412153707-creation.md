## Initial Notes

TamSession content-addressing of intermediates is one of the load-bearing uses of bit-exactness (it's why the meta-goal claim is worth the cost). Current framing: hash the output bytes bitwise, use the hash as a cache key. Alternative: Merkle tree of the computation graph, where each node's hash incorporates its op + inputs + parameters + precision + strategy. Merkle-based keys are STRUCTURAL (same computation = same key even if bytes differ slightly) vs BITWISE (same bytes = same key). Merkle permits invalidation patterns bitwise can't (change one op far down the graph, invalidate only its downstream nodes). Bitwise permits cross-machine sharing without needing to agree on the computation structure. Different trade-offs. Not investigated. Nobody has asked the question yet. Probably becomes interesting when Peak 6 reductions land and the session cache starts growing.
