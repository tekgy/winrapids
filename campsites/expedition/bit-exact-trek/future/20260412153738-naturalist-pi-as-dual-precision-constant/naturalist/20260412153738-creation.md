## Initial Notes

Math-researcher note: Cody-Waite reduction for sin/cos requires pi-over-two split into hi/lo dual-precision f64. The standard split is pi/2 = pi2_hi + pi2_lo where pi2_hi has its low 32 bits zeroed so 2*k*pi2_hi is exact for k up to 2^20. Beyond that you need Payne-Hanek (other campsite). Some implementations use a THREE-piece split for better coverage (pi2_hi + pi2_mid + pi2_lo) which extends the exact-reduction range. Worth checking: does math-researcher's sin-cos-design.md use 2-piece or 3-piece split? If 2-piece, the trigger for Payne-Hanek is earlier (around |x| > 2^20). If 3-piece, it's later. Minor point but the constant values should be documented in the coefficient tables as hex literals.
