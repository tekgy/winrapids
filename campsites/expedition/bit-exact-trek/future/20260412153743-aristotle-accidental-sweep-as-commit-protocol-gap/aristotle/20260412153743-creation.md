## Initial Notes

Navigator flagged the 'git add . accidentally sweeps files from other teammates' working space' pattern to pathmaker this session. This coordinates with my e05d495 observation (f64 phase doc swept into RFA commit by accident). The pattern is a commit-protocol gap: the team lacks a standard for 'how do you ensure only YOUR files go into YOUR commit when the working tree has uncommitted files from multiple roles?' Possible fixes: (a) targeted 'git add <specific-file>' — navigator's current answer, works but requires discipline; (b) role-specific working directories that git diff/status can filter — structural answer; (c) pre-commit hooks that warn on cross-role file mixes — automated answer; (d) git worktrees per role — heavy but clean. Worth a small design pass if the pattern repeats across sessions. Not urgent. Navigator is already carrying the discipline answer.
