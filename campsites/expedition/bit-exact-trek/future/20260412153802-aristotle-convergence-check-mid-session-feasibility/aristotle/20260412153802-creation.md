## Initial Notes

Practice file (~/.claude/practices/convergence-check.md) says 'run at garden transitions' — specifically before entering the garden or returning from it. My one run this session was at session-end, which is effectively a session-boundary and worked well. Open question: does the method work mid-session too? Hypothesis: YES, if (a) there are 3+ parallel outputs to compare, and (b) the outputs are 'hardened enough' that their structure is legible. Fresh half-thoughts probably don't have enough shape to rhyme against each other. Session-end worked because each of my deconstructions was locked at v1 or later. Could be tested: run a mid-session convergence check on a smaller input set (e.g., three Phase 2 irreducible-truth enumerations across deconstructions running in parallel, BEFORE any Move is committed). If the method still produces findings, mid-session runs are valuable. If the findings are too vague, the session-boundary rule is load-bearing. Worth one experiment.
