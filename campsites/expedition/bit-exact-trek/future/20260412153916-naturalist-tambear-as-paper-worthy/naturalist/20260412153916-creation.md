# naturalist-tambear-as-paper-worthy

Created: 2026-04-12T15:39:16-05:00
By: naturalist

## Initial Notes

Observation: tambear as currently scoped (bit-exact cross-backend f64 compiler with own IR, own libm, own PTX assembler, own SPIR-V assembler, own RFA, own CPU interpreter, all from first principles, all open-source) is literally a paper. The architectural novelty is the cross-backend bit-exact claim — no open-source project to my knowledge has publicly committed to and formally verified that claim before. The method (three preconditions from the guarantee-ledger, a named invariant protecting each, a compositional claim statement) is a reproducible framework other compiler projects could adopt. Worth asking: does Tekgy want this written up as a paper? ARITH 2027, SIGARCH, PLDI, SC, or even a CAV / TACAS venue for the formal-methods angle. The trek as-is is ~10 months from 'we have code' to 'we have a paper-ready result.' The work is there; the question is whether Tekgy wants the publication path vs the tools-first-paper-later path.
