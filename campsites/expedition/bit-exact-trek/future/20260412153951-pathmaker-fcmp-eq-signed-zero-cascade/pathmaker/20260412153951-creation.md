## Initial Notes

fcmp_eq(+0,-0)=true bites THREE separate design docs (sin-cos B2, tan B1, and implicitly log B1). This is the same bug shape in every odd function's zero fast-path. Pathmaker should add a lint rule: any 'if x == +0: return +0 / if x == -0: return -0' pattern is a bug. The correct idiom is always 'if x == 0: return x'.
