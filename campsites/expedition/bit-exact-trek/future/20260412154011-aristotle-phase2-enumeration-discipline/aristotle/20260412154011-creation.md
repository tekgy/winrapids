## Initial Notes

Methodological lesson from my f64 Phase 2 gap (I missed the composed-ULP-budget truth; navigator caught it; I acknowledged). Phase 2 (irreducible truths) is where Aristotle's method is most vulnerable to the 'I missed a role' bug. The enumeration discipline needs a concrete forcing function. Candidate: for every Phase 2, run a concrete example through the enumerated truths and ask 'does this example exercise every truth?' If a truth is unused for any example I can construct, it's redundant. If an example has no truth covering it, a truth is missing. The concrete-example forcing function is adjacent to how writers catch plot holes: walk through every scene and ask 'is this motivated?' Applied to Phase 2: walk through a concrete recipe (variance, pearson_r) and ask 'does each truth explain a constraint I actually observe?' If any constraint is unexplained, enumerate that truth. Small methodology upgrade. Could become a checklist step in future deconstructions.
