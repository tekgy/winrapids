Created: 2026-04-12T15:40:16-05:00

## Initial Notes

In team sessions with many parallel campsite sprays, the git commit pattern matters: the index.lock error appears repeatedly when multiple agents try to commit concurrently. Better protocol: collect a batch of untracked dirs, commit once per minute from pathmaker's side; let other agents queue rather than conflict. The 'git status | grep ^?? | xargs git add' rolling pattern is the right idiom.
