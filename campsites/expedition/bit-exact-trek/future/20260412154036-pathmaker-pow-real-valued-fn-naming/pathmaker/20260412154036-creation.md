## Initial Notes

pow-design.md Amendment 2 introduced real_valued_pow(a, b) as a named helper. But this name collides conceptually with 'pow for real (non-complex) inputs' which is ALL of tam_pow. Better name: positive_pow(a, b) or pow_via_exp_log(a, b). The function's distinguishing property is not that it's 'real-valued' but that it requires a > 0 and uses the exp(b*log(a)) path.
