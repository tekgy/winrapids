## Initial Notes

The .tam text format is the program text that makes op ordering explicit and prevents reassociation. But the interpreter also runs the ops. Pathmaker should add a test: 'print a .tam program to text, parse it back, run it, verify bit-identical results.' If round-trip print→parse→run changes anything, either the printer is lossy or the parser is — both are bugs. This is the text format's correctness test.
