## Initial Notes

Observational note about the first live test of the convergence-check methodology (team-lead ran it on VB-001..005 producing the Bucket A/B finding): the check worked because the five VB entries were REAL parallel outputs from a single team member (scout) investigating the same topic (Vulkan vendor ambiguities) with the same method (read spec, test device, compare to intent). The convergence was not an artifact of stylistic similarity — the entries had genuine structural content to compare. Interpretation: convergence-check works best when the corpus is 'parallel outputs from the same investigative method applied to different objects.' Where it might NOT work: (a) outputs from different methods on the same object (they share the object but not the method), (b) outputs from the same method on objects at different abstraction levels (they share the method but the grain mismatches). Worth a methodology refinement note for the practice file: 'convergence-check corpus selection criteria.'
