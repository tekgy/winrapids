## Initial Notes

fcmp_eq / fcmp_lt / fcmp_gt all have IEEE-754 semantics but PTX has setp.eq.f64, setp.lt.f64, etc. AND setp.equ.f64 (which treats NaN as equal) vs setp.eq.f64 (ordered). We must use the ORDERED variants (setp.eq.f64, not setp.equ.f64) for IEEE 754 compliance. The backend selection table needs to document: ordered vs unordered PTX predicates, which .tam IR ops map to which.
