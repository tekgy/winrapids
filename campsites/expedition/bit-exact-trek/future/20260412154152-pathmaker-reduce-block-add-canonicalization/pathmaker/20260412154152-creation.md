## Initial Notes

reduce_block_add.f64 in its sequential_left form does fold(0.0, x_0, x_1, ...). The NaN-semantics campsite I created earlier notes fold(0.0, NaN, ...) = NaN propagates. But: fold(NaN, 0.0) = NaN also propagates because the FIRST operand is NaN. Sequential_left means the accumulator carries NaN forward. What if all elements are 0 and only the initial is NaN? Then result = NaN. Is that the desired contract?
