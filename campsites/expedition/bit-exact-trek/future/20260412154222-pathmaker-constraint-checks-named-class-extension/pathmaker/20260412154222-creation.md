## Initial Notes

constraint_checks.toml uses constraint='nonzero_subnormal_positive'. Current NamedConstraint enum has this one class. But we need: 'subnormal' (any), 'finite' (not inf/nan), 'infinite_positive', 'infinite_negative', 'nan' (any), 'signed_zero_positive', 'signed_zero_negative'. The enum should be the full IEEE-754 class taxonomy. Adversarial will need all of these for oracle coverage of every transcendental's special-value matrix.
