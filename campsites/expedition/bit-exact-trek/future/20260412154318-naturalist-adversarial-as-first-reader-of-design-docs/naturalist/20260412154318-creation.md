## Initial Notes

Observation about adversarial's workflow: they're not waiting for implementation to critique — they're running the critique against design docs while the ink is still wet. This makes adversarial THE FIRST READER of every design doc in Peak 2 so far. From the team's perspective, this means adversarial's reading is the reality check: if adversarial's read can't find a blocker, the design is probably ready to implement. If adversarial finds 5 blockers (as they did with exp-design), the design isn't ready. Worth naming as a quality gate: 'design doc merge criterion is adversarial sign-off, not navigator sign-off' (for libm designs specifically). Navigator's sign-off is for the accuracy target and overall approach; adversarial's sign-off is for the per-function specification. Both are needed. Two-column signoff table would capture it.
