# naturalist-test-as-pinning-vs-testing

Created: 2026-04-12T15:43:49-05:00
By: naturalist

## Initial Notes

Observation about adversarial's test philosophy: their failing tests are not 'the code is broken' — they're 'the CORRECT behavior is X, the code currently produces Y, fix the code to produce X, do NOT adjust the assertion.' This is the pinning-test pattern. Every failing adversarial test is a captured spec fragment. If the team loosens the assertion to make the test pass, they've lost the spec. Worth naming as a team practice: 'adversarial's tests are pinning tests, not code-quality tests.' Adversarial's Peak 4 broadcast said it explicitly: 'Do not fix them by adjusting the assertion.' Worth a short methodology note so this doesn't get forgotten when someone non-adversarial tries to fix a red test by relaxing it.
