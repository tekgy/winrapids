## Initial Notes

Reading / research thread: the musl libm fork rationale and its subsequent bug-fix commit history. musl was forked from fdlibm in the 2010s because glibc's libm had accumulated modifications and bugs; musl wanted a clean base. Musl's git history therefore contains a curated trail of which fdlibm bugs were caught and fixed during cleanup, and which new bugs were introduced and fixed subsequently. Reading the commit log with 'git log --oneline src/math/' on musl would produce an annotated history of fdlibm bug lineage. Valuable for the libm lineage doc extension campsite (naturalist-libm-lineage-fuller-version). Could also become adversarial test cases: every historical bug becomes a test input to verify tambear doesn't repeat it.
