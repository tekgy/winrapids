# using-two-scopes

Created: 2026-04-10T16:45:14-05:00
By: math-researcher

## Initial Notes

DESIGN DECISION: using() consumed at Level 0-1 (primitives choose own params), persistent at Level 2-3 (pipelines impose consistency). step.with(n_bins=50) for step-specific. using(alpha=0.01) for analysis-level. Source: math-researcher + naturalist + aristotle convergence.

---

## Resolution (2026-04-10, scout + navigator)

The Level 0-1 / Level 2-3 framing is superseded by a cleaner three-tier model.

**Three-tier priority chain (highest to lowest):**

1. **Step-level `using()`** — consumed per-step, drain behavior unchanged.
   `correlation(x,y).using(method="pearson")` applies to that step only.
   Chain syntax makes scope unambiguous. No leakage.

2. **Session-level `session.defaults(alpha=0.01)`** — persists whole session,
   new concept, does NOT drain. User sets a principle once; all steps inherit.
   Analogous to `matplotlib.rcParams` or R's `options()`.

3. **Coded defaults** — our opinion, lowest priority. `unwrap_or(0.05)`.

**Lookup pattern:**
```rust
let alpha = using_bag.get_f64("alpha")                   // step-level
    .or_else(|| session.defaults.get_f64("alpha"))       // session-level
    .unwrap_or(0.05);                                    // coded default
```

**Implementation requirements:**
- Add `session_defaults: UsingBag` field to `TamSession`
- Add `session.defaults(...)` step handler to tbs_executor (or `set_defaults(...)`)
- All bypass wiring (V-03 through V-13) uses three-tier pattern, not two-tier

**Why this is correct:**
- Consumed-per-step eliminates chain ambiguity (step reads exactly its own block)
- Session defaults solve the "I want alpha=0.01 everywhere" use case without leakage
- Specific beats general beats opinion — clean precedence, no conflicts
- Additive change: no existing behavior changes, new capability added

Source: scout analysis of tbs_executor drain architecture + navigator synthesis.
