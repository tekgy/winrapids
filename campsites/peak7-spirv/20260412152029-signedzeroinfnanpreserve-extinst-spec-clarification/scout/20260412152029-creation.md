## Initial Notes

Open spec question: does SPV_KHR_float_controls SignedZeroInfNanPreserve execution mode cover OpExtInst (GLSL.std.450) or only core SPIR-V arithmetic ops? Spec text says it prevents optimizations assuming no NaN on arithmetic instructions. GLSL.std.450 extended instructions are technically OpExtInst not OpFAdd/OpFMul. VB-005 workaround (OpIsNan guard) makes this moot for I11 compliance, but the spec question is still open. Check SPV_KHR_float_controls errata or Vulkan spec clarifications before Peak 7 campsite 7.3.
