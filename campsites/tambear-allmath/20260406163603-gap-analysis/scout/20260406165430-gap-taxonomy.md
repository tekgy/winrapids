# tambear-allmath Gap Taxonomy
**All Math In All Fields — What We Don't Have Yet**

Created: 2026-04-06  
By: scout (informed by full audit of existing codebase)

---

## Methodology

Built from: (1) full audit of 68K lines / 38 modules in `crates/tambear/src/`, (2) systematic review of mathematical fields. For each domain: what algorithms matter most, and how they decompose to accumulate+gather.

Tambear's accumulate+gather taxonomy:
- **Kingdom A** = closed-form, one accumulate pass (linear algebra solves, weighted sums)
- **Kingdom B** = sequential scan, order-dependent (ARMA, Kalman, MCMC chains)
- **Kingdom C** = iterative/converging (EM, Newton-Raphson, optimization loops)
- **Kingdom D** = graph/recursive (tree traversals, dynamic programming)
- **Kingdom E** = multi-pass, pyramid (FFT-style, wavelet, hierarchical)

## 1. Pure Math Foundations

### 1.1 Abstract Algebra
**What's missing:** Groups, rings, fields, modules, ideals, quotient structures. `proof.rs` has algebraic *vocabulary* but no *computational* algebra.

| Algorithm/Operation | Kingdom | Decomposition |
|---|---|---|
| Group orbit + stabilizer | B | scatter by coset representatives, accumulate orbit elements |
| Polynomial GCD (Euclidean) | B | sequential division, remainder chain |
| Polynomial factoring (Berlekamp/Cantor-Zassenhaus) | C | iterate over factors modulo p |
| Galois field GF(p^n) arithmetic | A | accumulate via irreducible poly, mod-reduce |
| Smith/Hermite normal form | C | iterated column/row ops, Gaussian-style |
| Gröbner basis (Buchberger) | C | iterative S-polynomial reduction |
| Integer lattice (LLL reduction) | C | iterated basis reduction |
| Sylow theorems (computational) | D | subgroup search via DFS |

**Key types needed:**
```rust
struct GaloisField { p: u64, n: usize, irred: Vec<u64> }
struct Polynomial<F> { coeffs: Vec<F> }
struct Group { generators: Vec<Permutation>, order: usize }
```

**Accumulate decomposition highlight:** GF(p^n) multiplication = polynomial multiplication mod irred poly. The mod-reduce step is a single `accumulate(coeffs, range, mul_add_mod, reduce_mod_poly)`. The Berlekamp matrix is a tiled accumulate of Frobenius map applications.

### 6.1 Stochastic Processes ← HIGHEST PRIORITY
**Why highest:** WinRapids farms market signals. The market IS a stochastic process.

| Algorithm | Kingdom | Decomposition |
|---|---|---|
| Geometric Brownian Motion simulation | B | sequential: S_{t+1} = S_t · exp((μ-σ²/2)dt + σ√dt · Z) |

**The Hawkes process is a particularly glaring absence** given WinRapids markets. Trade arrivals, order flow, price jumps — all fit Hawkes. The intensity λ(t) = μ + Σ_i α·exp(-β(t-t_i)) is a scatter-accumulate over past events (decaying kernel). It's literally built for tambear's architecture.

## 7. Cross-Domain Structural Insights

### Things that look different but decompose identically

1. **Viterbi = Smith-Waterman = Knuth-Yao DP** — all are `accumulate(sequence, (max,+) semiring)`. The same abstract scan over a semiring. Different applications, identical decomposition.

2. **Belief propagation = marginalizing a factor graph** = `scatter(factor→variable messages) → accumulate(variable beliefs)`. Identical structure to LDPC decoding, Kalman filter update, and loopy BP in graphical models.

3. **Kalman filter predict step** = Gaussian product under linear transform = the same operation as a forward pass in a linear dynamical system. The accumulate is P_{t+1} = A·P_t·A' + Q — a quadratic form accumulate.

4. **All spectral methods** (DFT, DCT, WHT, Spherical Harmonics, Graph Laplacian eigenvectors) decompose as: `accumulate(basis_functions, inner_product_with_data, sum)`. They differ only in the basis. A single `spectral_transform(basis, data)` function with swappable basis would unify them all.

5. **ALL gradient-based optimizers** = `sequential_scan(parameters, (gradient, state), update_rule)`. Kingdom B, same structure. Adam/AdaGrad/RMSProp are just different `update_rule` closures. Already in tambear but worth noting the pattern.

6. **AES MixColumns = Reed-Solomon evaluation = BCH encoding** — all are polynomial evaluation over GF(2^m). The field multiplication is `accumulate(poly_coeffs, x^i, GF_mul_add)`.

### The missing Fock boundary in domain math

Domain-specific algorithms (neuroimaging GLM, genomics DP, seismic FD) all live at the Fock boundary in the same way tambear's financial signals do. Each domain's "raw tick" is: the voxel for fMRI, the nucleotide for genomics, the seismogram sample for seismology. The same Kingdom A/B accumulate structure applies universally — the domains just differ in their measurement type and physical model.
