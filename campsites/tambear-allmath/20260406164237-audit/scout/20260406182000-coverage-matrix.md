## The TBS Column — What's Actually Wired

`adversarial_tbs.rs` tests these operations through `tbs_executor::execute()`:

```
normalize()         → compute_engine / accumulate (z-score)
describe()          → descriptive (mean, variance, etc.)
skewness()          → descriptive
t_test()            → hypothesis
ks_test()           → hypothesis
anova()             → hypothesis
kmeans(k=N)         → knn (clustering)
knn(k=N)            → knn
pca()               → dim_reduction
train.linear()      → (linear regression module)
train.logistic()    → (logistic regression module)
discover_clusters() → knn/clustering
predict()           → model output
```

**31 modules have NO TBS wiring at all.** That includes: all of survival, bayesian, time_series, volatility, nonparametric, robust, information_theory, rng, graph, special_functions, series_accel, interpolation, signal_processing, spectral, spatial, physics, mixed_effects, panel, irt, factor_analysis, mixture, neural, multivariate, linear_algebra, optimization.

This is the most significant coverage gap — the .tbs scripting surface only exposes ~15% of the math library.
