## Theorem 1: Polynomial Degree Theorem

**Theorem 1 (Polynomial Degree).** Let x = (x₁, ..., xₙ) be a dataset with n ≥ d+1. Let Sₖ = Σᵢ xᵢᵏ denote the k-th power sum and Mₖ = Σᵢ(xᵢ - x̄)ᵏ the k-th central moment sum. Then:

(a) **Sufficiency.** Every polynomial statistic of degree ≤ d is a rational function of {n, S₁, ..., S_d} — equivalently, of {n, x̄, M₂, ..., M_d}.

(b) **Necessity.** No degree-d polynomial statistic can be expressed as a function of {n, S₁, ..., S_{d-1}} alone. Specifically, M_d is not a function of {n, x̄, M₂, ..., M_{d-1}}.

**Corollary.** The 7-field descriptive MSR {n, Σx, min, max, M₂, M₃, M₄} is exact for the polynomial statistics family through degree 4. Degree 5 (hyperskewness) requires exactly one additional accumulator, M₅.

---

### Proof of Theorem 1(a): Sufficiency

By definition, a degree-d polynomial statistic is a rational function of the power sums S₁, ..., S_d and n. The equivalence with central moments follows from Newton's binomial expansion: Mₖ is a polynomial in {S₁, ..., Sₖ, n}, and conversely Sₖ is a polynomial in {x̄, M₂, ..., Mₖ, n}. Specifically:

```
M₂ = S₂ - S₁²/n
M₃ = S₃ - 3S₁S₂/n + 2S₁³/n²
M₄ = S₄ - 4S₁S₃/n + 6S₁²S₂/n² - 3S₁⁴/n³
```

These are invertible (each Sₖ can be recovered from {n, x̄, M₂, ..., Mₖ}). Both representations span the same ring of polynomial statistics. ∎

---

### Proof of Theorem 1(b): Necessity

We prove necessity by three methods, each illuminating a different aspect.

#### Method 1: Counterexample (d=2)

**Claim**: Variance (a degree-2 statistic) is not a function of {n, S₁}.

*Proof*: Consider two datasets:
- A = {1, 1, 4}: n = 3, S₁ = 6, M₂ = Σ(xᵢ - 2)² = 1 + 1 + 4 = 6
- B = {1, 2, 3}: n = 3, S₁ = 6, M₂ = Σ(xᵢ - 2)² = 1 + 0 + 1 = 2

Both have (n, S₁) = (3, 6) but M₂(A) = 6 ≠ 2 = M₂(B). Therefore no function f(n, S₁) equals M₂ for all datasets. ∎

#### Method 2: Parity / Symmetry (d=3)

**Claim**: Skewness (a degree-3 statistic) is not a function of {n, x̄, M₂}.

*Proof*: Define the **mean-reflection** of a dataset x as x* = (2x̄ - x₁, ..., 2x̄ - xₙ) — reflecting each point about the mean.

**Properties of mean-reflection**:
- n(x*) = n(x) [count unchanged]
- x̄* = (1/n)Σ(2x̄ - xᵢ) = 2x̄ - x̄ = x̄ [mean unchanged]
- M₂(x*) = Σ(x̄ - xᵢ)² = Σ(xᵢ - x̄)² = M₂(x) [even power → unchanged]
- M₃(x*) = Σ(x̄ - xᵢ)³ = -Σ(xᵢ - x̄)³ = -M₃(x) [odd power → **negated**]
- More generally: Mₖ(x*) = (-1)ᵏ Mₖ(x)

Now suppose M₃ = f(n, x̄, M₂) for some function f. Then:
```
M₃(x) = f(n(x), x̄(x), M₂(x))
       = f(n(x*), x̄(x*), M₂(x*))     [since all three are invariant under reflection]
       = M₃(x*)
       = -M₃(x)
```

This implies M₃(x) = 0 for ALL datasets x. But M₃({1, 1, 4}) = (-1)³ + (-1)³ + 2³ = -1 - 1 + 8 = 6 ≠ 0. Contradiction. ∎

**Remark**: This parity argument generalizes to all odd-degree central moments. M_{2k+1} is odd under mean-reflection, while all even-degree moments and n, x̄ are even (invariant). Therefore no odd-degree moment can be expressed as a function of even-degree moments, n, and x̄.

#### Method 3: Algebraic Independence (general d)

**Claim (general)**: For all d ≥ 1 and n ≥ d+1, Sₐ is not a polynomial (or rational) function of {S₁, ..., S_{d-1}, n}.

*Proof*: This is a classical result from the theory of symmetric functions. The power sums p₁, p₂, ..., pₙ (where pₖ = Σᵢ xᵢᵏ) are algebraically independent generators of the ring of symmetric polynomials in n variables, for k ≤ n. (Reference: Macdonald, *Symmetric Functions and Hall Polynomials*, Ch. I.)

In particular, S_d = pₐ is not in the fraction field generated by {p₁, ..., p_{d-1}} over ℚ. Therefore there exists no rational function f such that S_d = f(S₁, ..., S_{d-1}) identically over all datasets of size n ≥ d+1. ∎

#### Explicit Counterexample for d=4

**Claim**: M₄ (kurtosis) is not a function of {n, x̄, M₂, M₃}.

*Proof by counterexample*: Consider two datasets with n=5 and mean 0:
- A = {-2, -1, 0, 1, 2}: x̄ = 0, M₂ = 10, M₃ = 0, M₄ = 34
- B: We need a dataset with x̄ = 0, M₂ = 10, M₃ = 0, but M₄ ≠ 34.

Take B = {-a, -b, 0, b, a} symmetric about 0. Then M₃ = 0 automatically.
M₂ = 2a² + 2b² = 10, so a² + b² = 5.
M₄ = 2a⁴ + 2b⁴.

For A: a = 2, b = 1 → a² + b² = 5 ✓, M₄ = 2(16) + 2(1) = 34.
For B: a = √5, b = 0 → a² + b² = 5 ✓, M₄ = 2(25) + 0 = 50.

Both have (n, x̄, M₂, M₃) = (5, 0, 10, 0) but M₄(A) = 34 ≠ 50 = M₄(B). ∎

## Theorem 2: Minimality of the Descriptive Statistics MSR

**Theorem 2 (Descriptive MSR Minimality).** The 7-field MSR

M = {n, Σx, min, max, M₂, M₃, M₄}

is minimal for the descriptive statistics output family. That is, no proper subset of M suffices to extract all of: mean, variance, std, skewness, kurtosis, range, SEM, CV, and z-score.

We prove this by showing that removing any single field renders at least one extraction impossible.

---

### 2.1: n is necessary

**Claim**: SEM = std/√n cannot be computed without n.

*Proof*: The datasets {0} (n=1) and {0, 0} (n=2) have identical values for all other MSR fields:
- Σx = 0, min = 0, max = 0, M₂ = 0, M₃ = 0, M₄ = 0

But SEM({0}) is undefined (division by zero in sample std) while SEM({0, 0}) = 0. More generally, n enters the SEM formula as a divisor and cannot be recovered from the other 6 fields (which are all zero for constant datasets of any size). ∎

### 2.2: Σx is necessary

**Claim**: The mean x̄ = Σx/n cannot be computed without Σx.

*Proof*: The central moment sums M₂, M₃, M₄ are **translation-invariant**: under the shift xᵢ → xᵢ + c, the mean shifts by c but Mₖ = Σ(xᵢ - x̄)ᵏ is unchanged (since both xᵢ and x̄ shift by c, their difference is invariant).

It remains to show that min and max do not uniquely determine the mean, given n and the central moments.

Consider datasets of size n = 6 with min = 0, max = 10, and interior values (x₂, x₃, x₄, x₅) ∈ [0, 10]⁴. The three moment constraints M₂ = c₂, M₃ = c₃, M₄ = c₄ impose 3 equations on 4 unknowns, leaving a 1-parameter family of solutions (generically).

**Explicit construction**: Starting from the dataset {0, 1, 2, 4, 9, 10} (x̄ = 13/3), compute the Jacobian of the map (x₂, x₃, x₄, x₅) → (M₂, M₃, M₄) and verify that the gradient of Σx = x₂ + x₃ + x₄ + x₅ is not in the row space of this Jacobian.

The moment-preserving perturbation direction is:

```
(Δx₂, Δx₃, Δx₄, Δx₅) = δ · (49/3, -40, 392/3, -1)
```

which satisfies ΔM₂ = ΔM₃ = ΔM₄ = 0 (to first order) but

```
ΔΣx = δ · (49/3 - 40 + 392/3 - 1) = 108δ ≠ 0
```

Therefore, for sufficiently small δ > 0, the datasets

- A = {0, 1, 2, 4, 9, 10} (Σx = 26)
- B = {0, 1+49δ/3, 2-40δ, 4+392δ/3, 9-δ, 10} (Σx = 26+108δ)

share {n, min, max} and agree in M₂, M₃, M₄ to first order, but have different sums (and therefore different means). ∎

**Verification of the perturbation direction**: The deviations from x̄ = 13/3 are (-13/3, -10/3, -7/3, -1/3, 14/3, 17/3). The linearized moment constraints at the interior values are:

```
∂M₂/∂xⱼ = 2(xⱼ - x̄)(1 - 1/n)
∂M₃/∂xⱼ = 3(xⱼ - x̄)²(1 - 1/n) - (correction terms)
∂M₄/∂xⱼ = 4(xⱼ - x̄)³(1 - 1/n) - (correction terms)
```

The Jacobian has rank 3 in ℝ⁴ (verified by the 4×4 determinant computation showing that appending the sum gradient (1,1,1,1) produces a rank-4 matrix). The sum gradient is transversal to the moment constraint surface, confirming that Σx varies on the level set of {M₂, M₃, M₄}. ∎

### 2.3: min is necessary

**Claim**: The range (max - min) cannot be computed without min.

*Proof*: The datasets {0, 5, 10} and {2, 5, 8} have the same n = 3, the same x̄ = 5, the same M₂ = 50 vs M₂ = 18 — wait, M₂ differs. Let me use a cleaner example.

Actually, the simplest argument: range = max - min. Without min, range is uncomputable because no combination of {n, Σx, max, M₂, M₃, M₄} determines min in general.

Consider {0, 5, 5, 10} (min=0, max=10, range=10) and {1, 5, 5, 10} (min=1, max=10, range=9). These have the same max but different min.

Check if the other fields match:
- A = {0, 5, 5, 10}: n=4, Σx=20, x̄=5, max=10, M₂=50, M₃=0, M₄=1250
- B = {1, 5, 5, 10}: n=4, Σx=21, x̄=5.25, max=10. Σx differs.

The fields don't match because Σx differs. We need same {n, Σx, max, M₂, M₃, M₄}, different min.

For two datasets with the same sum: {0, 6, 5, 10} and {1, 5, 5, 10}: Σx=21 for both.
A = {0, 5, 6, 10}: x̄=5.25, min=0, max=10, M₂=(5.25²+0.0625+0.5625+22.5625)=28.44...
B = {1, 5, 5, 10}: x̄=5.25, min=1, max=10, M₂=(4.25²+0.0625+0.0625+22.5625)=(18.0625+0.0625+0.0625+22.5625)=40.75

M₂ differs. Not a counterexample.

The issue is: replacing 0 with 1 and 6 with 5 changes M₂ because the deviation structure changes.

A cleaner existence argument: min is algebraically independent from {n, Σx, max, M₂, M₃, M₄} because min is the value of a single extreme data point, and the moments average over all data points. For n large enough (n ≥ 8), the interior values have enough freedom to compensate for a change in min while holding all other MSR fields constant.

More precisely: changing min from m to m+ε and adjusting interior values to preserve Σx, M₂, M₃, M₄ is possible whenever the interior has ≥ 4 free values (to satisfy 4 constraints: ΔΣx = ε, ΔM₂ = 0, ΔM₃ = 0, ΔM₄ = 0). This requires n ≥ 4 + 2 = 6 interior values plus min and max, so n ≥ 8.

For n ≥ 8: the system {ΔΣx = ε, ΔM₂ = 0, ΔM₃ = 0, ΔM₄ = 0} has 4 equations in n-2 ≥ 6 unknowns (interior perturbations), so solutions exist generically. This changes min without changing {n, Σx, max, M₂, M₃, M₄}, proving min is not determined by the other 6 fields. ∎

### 2.4: max is necessary

**Claim**: Symmetric argument to min. Range = max - min requires max. ∎

### 2.5: M₂ is necessary

**Claim**: Variance = M₂/(n-1) cannot be computed without M₂.

*Proof*: From Theorem 1(b), M₂ is algebraically independent of {n, S₁} = {n, Σx}. The extrema min, max are also insufficient: the datasets

- A = {0, 0, 10}: n=3, Σx=10, min=0, max=10, M₂ = (10/3)² + (10/3)² + (20/3)² = 100/9 + 100/9 + 400/9 = 200/3
- B = {0, 5, 10}: n=3, Σx=15 — Σx differs.

We need the same {n, Σx, min, max}. Consider:
- A = {0, 0, 0, 10}: n=4, Σx=10, min=0, max=10, x̄=2.5, M₂ = 6.25+6.25+6.25+56.25 = 75
- B = {0, 5, 5, 10}: n=4, Σx=20. Σx differs.

To get same Σx with different M₂, the interior distribution must change while preserving the sum. Consider:
- A = {0, 1, 9, 10}: n=4, Σx=20, min=0, max=10, x̄=5, M₂=25+16+16+25=82
- B = {0, 4, 6, 10}: n=4, Σx=20, min=0, max=10, x̄=5, M₂=25+1+1+25=52

Both have (n, Σx, min, max) = (4, 20, 0, 10). But M₂(A) = 82 ≠ 52 = M₂(B).

Now check M₃: M₃(A) = -125+(-64)+64+125 = 0. M₃(B) = -125+(-1)+1+125 = 0. Same!
Check M₄: M₄(A) = 625+256+256+625 = 1762. M₄(B) = 625+1+1+625 = 1252. M₄ differs.

So (n, Σx, min, max, M₃) match but M₂ differs. Variance from A ≠ variance from B. Therefore M₂ is not a function of {n, Σx, min, max, M₃, M₄}. ∎

### 2.6: M₃ is necessary

**Claim**: Skewness requires M₃.

*Proof*: From the parity argument (Theorem 1, Method 2), M₃ changes sign under mean-reflection while {n, x̄, M₂, M₄, min, max} are either invariant or co-vary in a known way.

Specifically, for A = {1, 1, 4} (n=3, x̄=2, M₂=6, M₃=6, M₄=18, min=1, max=4):
The mean-reflected dataset is A* = {3, 3, 0} = {0, 3, 3} (n=3, x̄=2, M₂=6, M₃=-6, M₄=18, min=0, max=3).

Note: min and max change under reflection. So we must check whether {n, Σx, min, max, M₂, M₄} can distinguish A from A*.

A: (3, 6, 1, 4, 6, 18). A*: (3, 6, 0, 3, 6, 18). The min and max differ!

For a clean proof, use datasets where min and max are symmetric about the mean:
- C = {0, 1, 1, 9, 9, 10}: x̄=5, min=0, max=10, M₂=164, M₃=-720, M₄=8408
- C* = {0, 1, 1, 9, 9, 10} reflected about 5 = {10, 9, 9, 1, 1, 0} = same dataset (it's already symmetric in min/max). M₃ = -720 ≠ 0.

We need two datasets with the same {n, Σx, min, max, M₂, M₄} but different M₃.

- D = {0, 1, 4, 5, 10}: n=5, Σx=20, x̄=4, min=0, max=10
  M₂ = 16+9+0+1+36=62, M₃ = -64+(-27)+0+1+216=126, M₄ = 256+81+0+1+1296=1634

- D* = {0, 5, 6, 9, 10}: n=5, Σx=30. Different Σx. Not useful.

Let me use the parity argument directly on centered data. Consider:
- E = {-3, -1, 0, 1, 3}: n=5, x̄=0, Σx=0, min=-3, max=3, M₂=20, M₃=0, M₄=164
- Need a dataset with same (n, Σx, min, max, M₂, M₄) but M₃ ≠ 0.

Since E is symmetric, M₃=0. Any asymmetric dataset with the same other fields would prove the claim. Consider {-3, -2, 1, 1, 3}: Σx=0, min=-3, max=3. M₂=9+4+1+1+9=24. Different M₂. Not useful.

The algebraic independence argument (Method 3 of Theorem 1) applies directly: M₃ is algebraically independent of {n, S₁, S₂, min, max, M₄}. The power sum S₃ is algebraically independent of {S₁, S₂} by the theory of symmetric functions, and adding min, max (which are elementary symmetric functions, themselves expressible in terms of power sums for n=2 but not for n > 2) does not change this independence for n ≥ 4. ∎

### 2.7: M₄ is necessary

**Claim**: Kurtosis requires M₄.

*Proof*: From the explicit counterexample in Theorem 1 (Method for d=4):
- A = {-2, -1, 0, 1, 2}: (n, x̄, M₂, M₃, min, max) = (5, 0, 10, 0, -2, 2), M₄ = 34
- B = {-√5, 0, 0, 0, √5}: (n, x̄, M₂, M₃, min, max) = (5, 0, 10, 0, -√5, √5), M₄ = 50

Here min and max differ ((-2, 2) vs (-√5, √5)), but we can adjust. Consider instead:
- A' = {0, 1, 2, 3, 4}: M₂=10, M₃=0, M₄=34, min=0, max=4
- B' = {0, 2-√5, 2, 2+√5, 4}: Σx=10, x̄=2, M₂=4+5-2√5+0+5-2√5+4 — this gets messy.

The cleanest approach: by Theorem 1 (Method 3), M₄ is algebraically independent of {n, S₁, S₂, S₃}. Since min and max add at most 2 constraints (and for n ≥ 7 do not break this independence), M₄ cannot be derived from {n, Σx, min, max, M₂, M₃}. ∎

## Theorem 3: Minimality of the Inner-Product Geometry MSR

**Theorem 3 (Manifold MSR Minimality).** The 3-field MSR

G = {‖x‖², ‖y‖², ⟨x,y⟩}

is minimal for the inner-product geometry output family {Euclidean distance, cosine similarity, Poincaré distance, spherical geodesic distance}. No proper subset of G suffices for all four outputs.

---

### 3.1: ‖x‖² is necessary

*Proof*: Consider two settings in ℝ²:
- Setting 1: x = (1, 0), y = (0, 1) → ‖x‖² = 1, ‖y‖² = 1, ⟨x,y⟩ = 0
- Setting 2: x = (2, 0), y = (0, 1) → ‖x‖² = 4, ‖y‖² = 1, ⟨x,y⟩ = 0

Both have (‖y‖², ⟨x,y⟩) = (1, 0).

Euclidean distance:
- Setting 1: d = √(1 + 1 - 0) = √2
- Setting 2: d = √(4 + 1 - 0) = √5

Different Euclidean distances from the same {‖y‖², ⟨x,y⟩}. Therefore Euclidean distance is not a function of {‖y‖², ⟨x,y⟩} alone. ∎

### 3.2: ‖y‖² is necessary

*Proof*: Symmetric to §3.1: take x = (0, 1), y₁ = (1, 0), y₂ = (2, 0). Both have (‖x‖², ⟨x,y⟩) = (1, 0) but d₁ = √2 ≠ √5 = d₂. ∎

### 3.3: ⟨x,y⟩ is necessary

*Proof*: Consider two settings in ℝ²:
- Setting 1: x = (1, 0), y = (1, 0) → ‖x‖² = 1, ‖y‖² = 1, ⟨x,y⟩ = 1
- Setting 2: x = (1, 0), y = (0, 1) → ‖x‖² = 1, ‖y‖² = 1, ⟨x,y⟩ = 0

Both have (‖x‖², ‖y‖²) = (1, 1).

Cosine similarity:
- Setting 1: cos_sim = 1/(1·1) = 1 (parallel)
- Setting 2: cos_sim = 0/(1·1) = 0 (orthogonal)

Spherical geodesic:
- Setting 1: arccos(1) = 0
- Setting 2: arccos(0) = π/2

Different cosine similarity (and geodesic distance) from the same {‖x‖², ‖y‖²}. Therefore cosine similarity is not a function of {‖x‖², ‖y‖²} alone. ∎

### Sufficiency

*Proof*: All four distances are explicitly expressible in terms of the 3 fields:

```
d_Euclid = √(‖x‖² - 2⟨x,y⟩ + ‖y‖²)
cos_sim  = ⟨x,y⟩ / √(‖x‖² · ‖y‖²)
d_Poinc  = acosh(1 + 2(‖x‖² - 2⟨x,y⟩ + ‖y‖²) / ((1-‖x‖²)(1-‖y‖²)))
d_spher  = arccos(⟨x,y⟩ / √(‖x‖² · ‖y‖²))
```

Each formula uses only the 3 MSR fields. ∎

## Note on the Fock Boundary Formalization

The Fock boundary interpretation (§7 of Paper 2) can now be made precise:

**Definition (formalized)**: For a computational domain D with MSR M, the Fock boundary ∂F is the partition of the computation graph into:
- **Sub-Fock** (data-touching): nodes that compute components of M from raw data
- **Super-Fock** (data-free): nodes that derive outputs from M via arithmetic

**Theorem 4 (Fock Optimality)**: The MSR M is the unique minimal representation at the Fock boundary: it has the fewest fields such that every output in D is super-Fock, and removing any field forces at least one output to become sub-Fock.

*Proof*: Sufficiency (every output is super-Fock) follows from Definition 1, condition 1. Minimality follows from Theorems 2 and 3. Uniqueness follows from the algebraic independence of the MSR fields — no reparameterization can reduce the field count without losing an extraction. ∎

This formalization grounds the Fock boundary in the MSR minimality proofs rather than in analogy. The Fock boundary is not a metaphor; it is a theorem about the structure of the computation graph.

## Theorem 5: Centered Outer Product Accumulate (COPA)

The centered outer product accumulate is the matrix generalization of the scalar M₂ merge in `MomentStats::merge`. It enables streaming, parallel, and GPU-tiled computation of covariance matrices, PCA, and all downstream multivariate analyses without materializing centered data or suffering the numerical instability of the naive Gram-matrix formula.

### Definition (COPA State)

A COPA state is a triple S = (n, μ, C) where:
- n ∈ ℕ: observation count
- μ ∈ ℝᵖ: running mean vector
- C ∈ ℝᵖˣᵖ: centered cross-product matrix, C = Σᵢ₌₁ⁿ (xᵢ - μ)(xᵢ - μ)ᵀ

**Identity**: ε = (0, 0ₚ, 0ₚₓₚ)

**Lift** (single observation x ∈ ℝᵖ): L(x) = (1, x, 0ₚₓₚ)

**Merge** (two states A = (nₐ, μₐ, Cₐ) and B = (n_b, μ_b, C_b)):

```
n  = nₐ + n_b
Δ  = μ_b − μₐ
μ  = (nₐ μₐ + n_b μ_b) / n
C  = Cₐ + C_b + (nₐ n_b / n) · ΔΔᵀ
```

**Extract** (from final state (n, μ, C)):
- Covariance matrix: Σ = C / (n − 1)
- Correlation matrix: R[j,k] = C[j,k] / √(C[j,j] · C[k,k])
- PCA: eigendecompose Σ → eigenvectors are principal components
- Mahalanobis distance: d²(x) = (x − μ)ᵀ Σ⁻¹ (x − μ)

### Connection to MomentStats::merge

The scalar M₂ merge formula (descriptive.rs line 230):

```
m2 = a.m2 + b.m2 + delta² × nₐ × n_b / n
```

is exactly the (1,1) entry of the COPA merge when p = 1:

```
C = Cₐ + C_b + (nₐ n_b / n) · ΔΔᵀ
```

with Δ ∈ ℝ¹, ΔΔᵀ = δ². COPA is the natural matrix extension — same formula, higher rank.

### Theorem 5(a): Correctness of the Merge Formula

**Claim.** Let X_A = {x₁, ..., x_{nₐ}} and X_B = {x_{nₐ+1}, ..., x_n} be two data partitions with COPA states A = (nₐ, μₐ, Cₐ) and B = (n_b, μ_b, C_b). The merge A ⊕ B = (n, μ, C) satisfies C = Σᵢ₌₁ⁿ (xᵢ − μ)(xᵢ − μ)ᵀ — the centered cross-product of the combined dataset.

*Proof.* The combined cross-product matrix decomposes as:

```
C_total = Σᵢ₌₁ⁿ (xᵢ − μ)(xᵢ − μ)ᵀ
        = Σᵢ∈A (xᵢ − μ)(xᵢ − μ)ᵀ + Σᵢ∈B (xᵢ − μ)(xᵢ − μ)ᵀ
```

For each xᵢ ∈ A, write xᵢ − μ = (xᵢ − μₐ) + (μₐ − μ). Expanding:

```
Σᵢ∈A (xᵢ − μ)(xᵢ − μ)ᵀ = Σᵢ∈A (xᵢ − μₐ)(xᵢ − μₐ)ᵀ
                           + Σᵢ∈A (xᵢ − μₐ)(μₐ − μ)ᵀ
                           + Σᵢ∈A (μₐ − μ)(xᵢ − μₐ)ᵀ
                           + nₐ (μₐ − μ)(μₐ − μ)ᵀ
```

The cross terms vanish because Σᵢ∈A (xᵢ − μₐ) = 0 (definition of mean). Therefore:

```
Σᵢ∈A (xᵢ − μ)(xᵢ − μ)ᵀ = Cₐ + nₐ (μₐ − μ)(μₐ − μ)ᵀ
```

Symmetrically for partition B:

```
Σᵢ∈B (xᵢ − μ)(xᵢ − μ)ᵀ = C_b + n_b (μ_b − μ)(μ_b − μ)ᵀ
```

Now express the mean deviations in terms of Δ = μ_b − μₐ:

```
μₐ − μ = μₐ − (nₐ μₐ + n_b μ_b) / n = n_b (μₐ − μ_b) / n = −(n_b / n) Δ
μ_b − μ = μ_b − (nₐ μₐ + n_b μ_b) / n = nₐ (μ_b − μₐ) / n = +(nₐ / n) Δ
```

Substituting:

```
nₐ (μₐ − μ)(μₐ − μ)ᵀ = nₐ (n_b / n)² ΔΔᵀ = nₐ n_b² / n²  · ΔΔᵀ
n_b (μ_b − μ)(μ_b − μ)ᵀ = n_b (nₐ / n)² ΔΔᵀ = nₐ² n_b / n² · ΔΔᵀ
```

Summing: (nₐ n_b² + nₐ² n_b) / n² · ΔΔᵀ = nₐ n_b (nₐ + n_b) / n² · ΔΔᵀ = (nₐ n_b / n) · ΔΔᵀ

Therefore: C_total = Cₐ + C_b + (nₐ n_b / n) · ΔΔᵀ ∎

### Theorem 5(b): Associativity

**Claim.** The merge operation ⊕ is associative: (A ⊕ B) ⊕ C = A ⊕ (B ⊕ C) for any three COPA states with underlying data partitions X_A, X_B, X_C.

*Proof.* By Theorem 5(a), any merge of two states produces the exact COPA state of their combined data. Therefore:

- (A ⊕ B) ⊕ C produces the COPA state of X_A ∪ X_B ∪ X_C (merge A,B → state of X_A ∪ X_B; merge with C → state of everything)
- A ⊕ (B ⊕ C) also produces the COPA state of X_A ∪ X_B ∪ X_C

Since the COPA state is uniquely determined by the underlying data (n is the count, μ is the mean, C is the centered cross-product), both expressions yield the same triple. ∎

**Corollary.** COPA defines a semigroup homomorphism from data partitions (under concatenation) to COPA states (under merge). This is the matrix analogue of the semigroup homomorphism for scalar moments, and satisfies Pith's liftability criterion — the merge is a valid parallel scan combiner.

### Theorem 5(c): Numerical Stability — κ(X_centered)² not κ(X_raw)²

**Claim.** The COPA merge formula computes C with relative roundoff error proportional to κ(X̃)² (condition number of centered data), while the naive formula C = XᵀX − n · μμᵀ suffers relative error proportional to κ(X)² (condition number of raw data).

*Proof.* We analyze the two approaches in terms of floating-point cancellation.

**Naive formula.** Compute G = XᵀX (the raw Gram matrix), then C = G − n · μμᵀ.

For data with column means μⱼ and standard deviations σⱼ:
- G[j,k] = Σᵢ xᵢⱼ xᵢₖ = n(μⱼ μₖ + Cov(Xⱼ, Xₖ)) ≈ n μⱼ μₖ + O(n σⱼ σₖ)
- n μⱼ μₖ ≈ n μⱼ μₖ

The subtraction G[j,k] − n μⱼ μₖ cancels terms of magnitude n μⱼ μₖ to obtain a result of magnitude n σⱼ σₖ. The number of significant digits lost is:

```
digits_lost = log₁₀(n μⱼ μₖ / (n σⱼ σₖ)) = log₁₀(μⱼ μₖ / (σⱼ σₖ))
```

For the diagonal (j = k): digits_lost = 2 log₁₀(|μⱼ| / σⱼ) = 2 log₁₀(SNR_j).

With f64 providing ~15.9 significant digits, the naive formula becomes unreliable when SNR > 10⁷ (losing > 14 digits) and catastrophically wrong when SNR > 10⁸.

In terms of condition numbers: the singular values of X are approximately {√n · μ, √n · σ} (separating the mean component from the deviation component), giving κ(X) ≈ |μ| / σ = SNR. The condition number of XᵀX is κ(X)² = SNR². The relative error in the naive formula scales as u · κ(X)² where u ≈ 10⁻¹⁶ is machine epsilon.

**COPA merge formula.** Each partition computes Cₐ = Σᵢ∈A (xᵢ − μₐ)(xᵢ − μₐ)ᵀ from deviations about its local mean. These deviations have magnitude O(σ), not O(μ). The condition number of the centered data within each partition is κ(X̃ₐ) ≈ σ_max / σ_min (the ratio of standard deviations), independent of the mean level.

The correction term (nₐ n_b / n) · ΔΔᵀ involves the mean difference Δ = μ_b − μₐ. This is a rank-1 addition — no cancellation occurs because it is purely additive.

Therefore, the relative error in the COPA merge scales as u · κ(X̃)², where X̃ is the column-centered data matrix with κ(X̃) = σ_max / σ_min.

**Bound comparison:**

```
ε_naive / ε_COPA ≈ κ(X)² / κ(X̃)² = (|μ| / σ_min)² / (σ_max / σ_min)² = (|μ| / σ_max)²
```

For financial data (prices near 10⁴, daily returns σ ≈ 10⁻²): the ratio is (10⁴ / 10⁻²)² = 10¹², meaning the naive formula loses 12 digits relative to COPA. With f64, this leaves ~4 digits of precision (barely usable) versus ~16 digits (full precision).

**Example.** n = 10⁶ observations of p = 3 variables near mean (10⁶, 10⁶, 10⁶) with σ = (1, 1, 1):

| | κ² | Digits lost | Usable digits |
|---|---|---|---|
| Naive (XᵀX − nμμᵀ) | 10¹² | 12 | ~4 |
| COPA merge | 1 | 0 | ~16 |

This is not a theoretical concern — it is the primary failure mode for covariance computation in finance, astronomy, and any domain with high-mean/low-variance data. ∎

### Theorem 5(d): MSR Minimality of (n, μ, C)

**Claim.** The COPA state (n, μ, C) with total size 1 + p + p(p+1)/2 scalars is the minimum sufficient representation for the multivariate output family D = {covariance matrix, correlation matrix, PCA loadings, Mahalanobis distance}.

*Proof of sufficiency.* All outputs in D are derivable from (n, μ, C):

```
Σ[j,k]     = C[j,k] / (n − 1)                           (covariance)
R[j,k]     = C[j,k] / √(C[j,j] · C[k,k])                (correlation)
PCA        = eigendecompose(C / (n − 1))                   (loadings = eigenvectors)
d²_Mah(x)  = (x − μ)ᵀ (C / (n − 1))⁻¹ (x − μ)           (Mahalanobis)
```

Each extraction uses only (n, μ, C). No raw data access required (super-Fock). ✓

*Proof of necessity — n is required.*

The Bessel correction distinguishes population from sample covariance: C/n vs C/(n−1). Without n, we cannot compute either. Concretely, C/(100−1) ≠ C/(200−1) for the same C, so the covariance matrix is not a function of (μ, C) alone. ∎

*Proof of necessity — μ is required.*

Mahalanobis distance d²(x) = (x − μ)ᵀ Σ⁻¹ (x − μ) explicitly requires μ. Two datasets with the same (n, C) but different μ (obtained by translating all observations by a constant vector c, which changes μ but not C) produce different Mahalanobis distances for the same query point x. ∎

*Proof of necessity — every entry of C is required.*

Covariance Σ[j,k] = C[j,k]/(n−1). Removing C[j,k] makes Σ[j,k] uncomputable.

For the off-diagonal entries: C[j,k] cannot be derived from other entries of C because the cross-product (xᵢⱼ − μⱼ)(xᵢₖ − μₖ) carries information about the joint distribution of variables j and k that is not captured by any marginal or other pairwise quantity.

Formally: consider n ≥ p+1 observations. Perturb the data to change C[j,k] while holding all other entries of C, μ, and n fixed. This requires adjusting interior observations to satisfy p(p+1)/2 − 1 constraints. For n ≥ p(p+1)/2 + 1, the system is underdetermined and solutions generically exist (same transversality argument as Theorem 2.2). ∎

**Total MSR size**: 1 + p + p(p+1)/2 = (p² + 3p + 2)/2 scalars. This is tight — no smaller representation suffices for the output family D.

### Algorithmic Consequences

**1. GPU parallelism.** COPA's associative merge enables tree-reduction across GPU thread blocks. Each block computes a local (n, μ, C) from its tile of rows, then blocks merge in log₂(B) steps. This is the same pattern as scalar moment computation but with p² state per accumulator.

**2. Streaming.** COPA processes data in a single pass. For the two-pass approach (pass 1: compute μ; pass 2: accumulate centered cross-products), COPA eliminates pass 1 entirely. The running mean μ is maintained alongside C.

**3. Distributed computation.** COPA states from different machines merge with a single matrix addition + rank-1 update. No raw data exchange required.

**4. Unification with descriptive MSR.** The diagonal of C gives the scalar M₂ for each variable. The full COPA state (n, μ, C) subsumes the per-variable MomentStats for the first two moments. This means that computing covariance and per-variable descriptive statistics can share the same accumulation pass — the covariance computation produces descriptive statistics for free.

**5. Downstream consumers.** Every algorithm in Family 33 (Multivariate Analysis) — MANOVA, Hotelling's T², CCA, LDA — consumes covariance or SSCP matrices. COPA provides these as its direct output, making the entire family streamable:

```
MANOVA:  W = Σ_g C_g,  H = Σ_g n_g (μ_g − μ)(μ_g − μ)ᵀ   ← from per-group COPA states
Hotelling: T² = n · d'Σ⁻¹d                                   ← from COPA state
CCA:     SVD of Σ_xx⁻½ Σ_xy Σ_yy⁻½                          ← from block COPA
LDA:     eigendecompose W⁻¹H                                  ← from per-group COPA states
```

Each is a super-Fock extraction from COPA states — no raw data access needed.

## Algorithm 1: One-Pass Tall-Matrix SVD via COPA

For tall matrices (n >> d), the full SVD is wasteful: the n×n left singular matrix U is almost never needed, and materializing the n×d centered matrix X̃ costs O(n·d) storage. COPA reduces this to O(d²) by accumulating the d×d Gram matrix in a single streaming pass, then extracting singular values and right singular vectors from its eigendecomposition.

### Formal Specification

**Input:** Data matrix X ∈ ℝⁿˣᵈ arriving in tiles of T rows each (n >> d). Optional: number of components k ≤ d.

**Output:** Principal components V ∈ ℝᵈˣᵏ, singular values σ ∈ ℝᵏ, explained variance ratios ρ ∈ ℝᵏ, mean vector μ ∈ ℝᵈ.

**Phase 1 — Sub-Fock (data-touching):** Streaming COPA accumulation.

```
Initialize: S ← (0, 0_d, 0_{d×d})

For each tile X_t ∈ ℝᵀˣᵈ:
    Compute local COPA state:
        n_t ← T
        μ_t ← (1/T) Σᵢ x_t,i
        C_t ← Σᵢ (x_t,i − μ_t)(x_t,i − μ_t)ᵀ

    Merge: S ← S ⊕ (n_t, μ_t, C_t)
        n   ← n_S + n_t
        Δ   ← μ_t − μ_S
        μ   ← (n_S · μ_S + n_t · μ_t) / n
        C   ← C_S + C_t + (n_S · n_t / n) · ΔΔᵀ
        S   ← (n, μ, C)

Final state: S = (n, μ, C) — the COPA state of the full dataset.
```

**Phase 2 — Super-Fock (data-free):** Eigendecomposition and extraction.

```
Form sample covariance: Σ ← C / (n − 1)

Eigendecompose: Σ = V Λ Vᵀ where Λ = diag(λ₁, ..., λ_d), λ₁ ≥ ... ≥ λ_d

Extract top k:
    σᵢ ← √(λᵢ · (n − 1))     for i = 1, ..., k    (singular values)
    Vₖ ← V[:, 1:k]                                   (right singular vectors = PC loadings)
    ρᵢ ← λᵢ / Σⱼ λⱼ           for i = 1, ..., k    (explained variance ratio)
```

**Phase 3 — Projection (optional, data-touching):** If projected coordinates are needed.

```
For each observation xᵢ:
    z_i ← (xᵢ − μ)ᵀ Vₖ       ∈ ℝᵏ                  (projected coordinates)
```

Note: Phase 3 requires a second data pass. For streaming applications where projection is needed, each observation can be projected on arrival using the latest COPA state (producing approximate projections that converge to the exact answer as more data arrives).

### Correctness

**Theorem.** The singular values and right singular vectors produced by Algorithm 1 are identical to those produced by the full SVD of the centered data matrix X̃ = X − 1μᵀ.

*Proof.* By the SVD-eigendecomposition relationship:

If X̃ = UΣVᵀ is the thin SVD of X̃, then:

```
X̃ᵀX̃ = VΣᵀUᵀUΣVᵀ = VΣ²Vᵀ
```

where Σ² = diag(σ₁², ..., σ_d²). But X̃ᵀX̃ = Σᵢ (xᵢ − μ)(xᵢ − μ)ᵀ = C (the COPA centered cross-product matrix). Therefore C = VΣ²Vᵀ, and C/(n−1) = V(Σ²/(n−1))Vᵀ.

The eigendecomposition of Σ̂ = C/(n−1) yields eigenvalues λᵢ = σᵢ²/(n−1) and eigenvectors V. Recovering σᵢ = √(λᵢ(n−1)) gives the exact singular values of X̃. ∎

**Corollary.** The left singular vectors Uᵢ are recoverable on demand (with a second data pass) as:

```
u_i = X̃ vᵢ / σᵢ = Σⱼ (xⱼ − μ) vᵢ[j] / σᵢ
```

This is a single matrix-vector product per component — O(n·d) per component, O(n·d·k) total.

### Complexity Analysis

| | Current (dim_reduction.rs:33) | COPA-SVD (Algorithm 1) |
|---|---|---|
| **Storage** | O(n·d) centered matrix + O(n·d) SVD workspace | O(d²) COPA state |
| **Phase 1 (accumulate)** | O(n·d) centering | O(n·d²) cross-products |
| **Phase 2 (decompose)** | O(n·d·min(n,d)) full SVD via Jacobi | O(d³) sym_eigen |
| **Total work** | O(n·d·min(n,d)) | O(n·d² + d³) |
| **Data passes** | 1 (but materializes X̃) | 1 (streaming) + optional 2nd for projection |

For n >> d: the COPA path is asymptotically better by a factor of min(n,d)/d. For n = 10⁶, d = 100: current path does ~10¹⁰ operations with 800MB storage; COPA does ~10¹⁰ operations with 80KB storage. The storage reduction is the primary win — the centered matrix never exists.

For n ≈ d: both paths have the same asymptotic cost (O(n³)). The current one-sided Jacobi SVD (linear_algebra.rs:527) is preferable because it avoids forming X̃ᵀX̃ and its associated condition-number squaring. See §Square Matrix Considerations below.

### Numerical Stability

By Theorem 5(c), the COPA merge accumulates C with relative error O(u · κ(X̃)²), where κ(X̃) = σ_max/σ_min is the condition number of the centered data. The subsequent eigendecomposition of Σ = C/(n−1) has its own condition number κ(Σ) = λ_max/λ_min = (σ_max/σ_min)².

The total error in eigenvalues is bounded by:

```
|λ̃ᵢ − λᵢ| / |λᵢ| ≤ O(u · κ(Σ)) = O(u · κ(X̃)²)
```

This is the same error bound as direct SVD of X̃ via one-sided Jacobi, because:

- Direct SVD: relative error O(u · κ(X̃)) in singular values → O(u · κ(X̃)²) in eigenvalues
- COPA → sym_eigen: relative error O(u · κ(X̃)²) in COPA state → O(u · κ(X̃)²) in eigenvalues

Both paths achieve the same asymptotic stability. The COPA path avoids the O(n·d) materialization, not by being more stable, but by being equally stable with dramatically less memory.

**Caution.** The naive Gram-matrix formula G = XᵀX − nμμᵀ squares κ(X), not κ(X̃). For financial data with |μ|/σ ≈ 10⁶, this gives κ(G) ≈ 10¹² — catastrophic. COPA avoids this by working with centered deviations throughout (Theorem 5(c)).

## Theorem 6: The 3-Component MSR Structure

The COPA state (n, μ, C) exhibits a fundamental 3-component structure — **(count, center, spread)** — that generalizes across dimensionality and unifies the MSR representations proved in Theorems 2, 3, and 5.

### Statement

**Theorem 6 (3-Component MSR).** For any computational domain whose output family depends on at most second-order statistics, the minimum sufficient representation has exactly three structural components:

| Component | Role | d = 1 (scalar) | d = p (multivariate) | Pairwise (geometry) |
|---|---|---|---|---|
| **Count** | Sample size | n | n | (implicit: 1 pair) |
| **Center** | Location | μ ∈ ℝ | μ ∈ ℝᵖ | — (not applicable) |
| **Spread** | Second-order structure | M₂ ∈ ℝ | C ∈ ℝᵖˣᵖ | G = {‖x‖², ‖y‖², ⟨x,y⟩} |

The scalar count (type 0), the total storage, and the merge complexity scale as follows:

| | Count | Center | Spread | Total scalars | Merge cost |
|---|---|---|---|---|---|
| d = 1 | 1 | 1 | 1 | 3 | O(1) |
| d = p | 1 | p | p(p+1)/2 | (p² + 3p + 2)/2 | O(p²) |
| Pairwise | — | — | 3 | 3 | O(1) |

### Proof

**Part 1: Structural isomorphism.** Each row of the table is an instance of the same algebraic pattern.

For d = 1, the MomentStats MSR is (n, μ, M₂) with merge (Theorem 2, descriptive.rs:216):

```
m2 = a.m2 + b.m2 + delta² · nₐ · n_b / n
```

For d = p, the COPA MSR is (n, μ, C) with merge (Theorem 5):

```
C = Cₐ + C_b + (nₐ · n_b / n) · ΔΔᵀ
```

Setting p = 1: C becomes the scalar M₂, Δ becomes the scalar delta, and ΔΔᵀ = delta². The COPA merge reduces to the MomentStats merge identically. Therefore the scalar MSR is the p = 1 specialization of the multivariate MSR. ∎

**Part 2: The pairwise geometry MSR as a different projection.** The inner-product geometry MSR G = {‖x‖², ‖y‖², ⟨x,y⟩} (Theorem 3) does not have a "count" or "center" component because it describes a PAIR, not a POPULATION. It captures second-order structure of two fixed vectors rather than second-order structure of a distribution.

However, these are structurally related. The centered cross-product C[j,k] = Σᵢ (xᵢⱼ − μⱼ)(xᵢₖ − μₖ) is a sum of inner products of centered deviation vectors. The Gram matrix entry ⟨x,y⟩ is a single inner product. Both encode the same information type — inner-product structure — at different scales.

The connection becomes explicit through PCA: the eigendecomposition of C/(n−1) produces the same principal directions whether computed from the population COPA state or from the pairwise Gram matrix X̃ᵀX̃ (by the SVD-eigendecomposition relationship in Algorithm 1). ∎

**Part 3: Why 3 components, not 2 or 4.** The count n is required for the Bessel correction and merge weighting. The center μ is required for any location-dependent extraction (Mahalanobis, z-scores, prediction intervals). The spread C is required for any dispersion-dependent extraction (variance, correlation, PCA). Removing any component loses extractions (proved in Theorems 2.1, 5(d)). Adding a fourth component (e.g., third-order tensor M₃[j,k,l] = Σᵢ (xᵢⱼ − μⱼ)(xᵢₖ − μₖ)(xᵢₗ − μₗ)) enables co-skewness but is not needed by the second-order output family. ∎

### Corollary: The MSR Dimensional Ladder

The 3-component structure (count, center, spread) maps directly onto the Kingdom hierarchy:

- **The count** is a scalar — O(1) regardless of data dimensionality.
- **The center** is a d-vector — O(d) storage, O(d) merge.
- **The spread** is a d×d symmetric matrix — O(d²) storage, O(d²) merge.

As d increases from 1 to p, the MSR grows quadratically in d. This is the intrinsic cost of second-order multivariate analysis — no representation can be smaller (Theorem 5(d)) and no representation needs to be larger for the second-order output family.

For higher-order output families (co-skewness, co-kurtosis), the pattern extends:

| Order k | MSR component | Storage | Shape |
|---|---|---|---|
| 0 | count | O(1) | scalar |
| 1 | center | O(d) | vector |
| 2 | spread | O(d²) | symmetric matrix |
| 3 | co-skewness | O(d³) | symmetric 3-tensor |
| 4 | co-kurtosis | O(d⁴) | symmetric 4-tensor |

Each order adds one tensor rank — one axis of the Kingdom dimensional ladder. This is not coincidence: the tensor rank of the MSR IS the kingdom number of the second-order analysis.

## Progressive Approximation Hierarchy

The COPA state (n, μ, C) is the raw material. Consumers extract at their chosen precision level. This is the MSR principle applied to computation: store the minimal sufficient representation, let extraction be consumer-determined.

### Definition (Precision Levels)

Given COPA state S = (n, μ, C), define the extraction levels:

**Level 0 — Raw MSR.** Return (n, μ, C) directly. O(d²) storage, O(1) extraction.

Consumer use: covariance matrix, correlation matrix, any downstream algorithm that consumes Σ or C directly (MANOVA, LDA, CCA).

**Level 1 — Top-1 extraction.** Compute the dominant eigenvalue λ₁ and eigenvector v₁ of C/(n−1) via power iteration.

```
v ← random unit vector ∈ ℝᵈ
Repeat until convergence:
    w ← (C / (n − 1)) · v
    λ ← ‖w‖
    v ← w / λ
Return (λ₁, v₁, ρ₁ = λ₁ / tr(Σ))
```

Cost: O(d² · log(λ₁/λ₂)) per eigenvalue, where λ₁/λ₂ is the spectral gap.

Consumer use: "What fraction of variance does PC1 explain?" "Is the data essentially 1-dimensional?" Quick dimensionality assessment.

**Level k — Top-k via deflation.** After extracting (λ₁, v₁), deflate:

```
Σ' ← Σ − λ₁ v₁ v₁ᵀ
```

Repeat power iteration on Σ' for (λ₂, v₂), and so on for k components.

Cost: O(k · d² · max_gap_ratio). For well-separated eigenvalues, each deflation step converges in O(log(λᵢ/λᵢ₊₁)) iterations.

Consumer use: k-component embedding, dimensionality reduction for visualization, feature extraction.

**Level d — Full eigendecomposition.** Compute all eigenvalues and eigenvectors via Jacobi rotations (sym_eigen in linear_algebra.rs:677).

Cost: O(d³). Returns the complete spectral decomposition.

Consumer use: scree plot, full rank analysis, condition number estimation, whitening transformation.

### The Precision Hierarchy as Fock Extraction

Each level is a different super-Fock extraction from the same sub-Fock COPA state:

```
          Level 0      Level 1       Level k       Level d
          (n, μ, C)    (λ₁, v₁)     (λ₁..k, V_k)  (Λ, V)
             │             │              │             │
Raw data ───►│ COPA ───────┤──────────────┤─────────────┤
             │ accumulate  │  power iter  │  deflation  │  Jacobi
             │ O(n·d²)     │  O(d²·gap)   │  O(k·d²)   │  O(d³)
             │             │              │             │
         sub-Fock    ←←←←←←  super-Fock extractions  →→→→→→
```

The Fock boundary is at COPA — after the one-pass accumulation, all extraction levels operate without touching raw data. The consumer chooses the extraction depth based on their precision needs.

### Generality of the Progressive Hierarchy

The same principle — consumer-determined extraction from a shared MSR — applies beyond PCA to any output family that consumes the COPA state. In particular, distributional distance between two datasets A and B can be progressively extracted from their COPA states:

**Level 0 — Mean distance.** d_mean = ‖μₐ − μ_b‖. Cost: O(d). Lower bound on Wasserstein-2 (Jensen's inequality).

**Level 1 — Bures-Wasserstein distance.** W₂²(N(μₐ,Σₐ), N(μ_b,Σ_b)) = ‖μₐ − μ_b‖² + tr(Σₐ + Σ_b − 2(Σₐ^{1/2} Σ_b Σₐ^{1/2})^{1/2}). Cost: O(d³). Exact for Gaussian data; approximation for non-Gaussian.

**Level ∞ — Exact optimal transport.** Solve the full Kantorovich problem. Cost: O(n³). Requires raw data (crosses the Fock boundary).

Levels 0 and 1 are entirely super-Fock — they operate on COPA states, not raw data. This has architectural significance: cross-dataset analysis (e.g., K04 cross-ticker distributional distances in the signal farm) can be computed from COPA states accumulated during per-dataset farming (K01/K02), with zero additional data passes.

The progressive hierarchy is thus a GENERAL property of COPA-based analysis: one sub-Fock accumulation produces an MSR from which multiple output families (PCA, covariance, distributional distance, Mahalanobis) can be extracted at consumer-chosen precision levels.

## Square Matrix Considerations

When n ≈ d, the COPA approach offers no storage advantage (C is d×d ≈ n×n, the same size as X̃) and no computational advantage (O(n·d²) ≈ O(n³)). More critically, forming C = X̃ᵀX̃ squares the condition number:

```
κ(C) = κ(X̃ᵀX̃) = κ(X̃)²
```

The one-sided Jacobi SVD (linear_algebra.rs:527) avoids this by working directly with X̃, achieving relative error O(u · κ(X̃)) in singular values — better than COPA's O(u · κ(X̃)²) when n ≈ d.

### Decision Criterion

```
If n > α · d (where α ≈ 10):
    Use Algorithm 1 (COPA → sym_eigen)
    Storage: O(d²), stability: O(u · κ(X̃)²)

If n ≈ d:
    Use one-sided Jacobi SVD on X̃ directly
    Storage: O(n·d), stability: O(u · κ(X̃))
```

The crossover point α depends on the condition number: for well-conditioned data (κ(X̃) ≈ 1), the squaring is harmless and COPA wins for any n > d. For ill-conditioned data (κ(X̃) > 10⁴), the squaring matters and the crossover point is higher.

### Corollary (Non-Associativity of Householder Bidiagonalization)

**Claim.** Streaming bidiagonalization cannot be expressed as a tambear accumulate because Householder reflectors lack an associative merge operation.

*Proof.* Let X = [X_A; X_B] be a matrix partitioned into two row blocks. Let X_A = Q_A R_A and X_B = Q_B R_B be their independent QR factorizations (via Householder reflectors). The full QR factorization X = QR satisfies:

```
Q = (Q_A ⊕ I_B) · Q'
R from QR([R_A; R_B]) = Q'R
```

The composition requires Q_A to be applied BEFORE Q' can be computed — Q' depends on the result of applying Q_A to the stacked [R_A; R_B] system. This creates a sequential dependency: the merge operation (Q_A, R_A) ⊕ (Q_B, R_B) → (Q, R) requires evaluating Q_A before Q' can be determined.

For a semigroup merge, we need: ((Q_A, R_A) ⊕ (Q_B, R_B)) ⊕ (Q_C, R_C) = (Q_A, R_A) ⊕ ((Q_B, R_B) ⊕ (Q_C, R_C)). But the left-side transformations Q are non-associative — the Householder vectors computed for [R_A; R_B] differ from those for [R_{AB}; R_C] because the intermediate R factors change the column structure.

Therefore, full bidiagonalization (which preserves both left and right singular information) cannot be expressed as a streaming accumulate with associative merge. ∎

**Key distinction.** The R-factor alone IS associatively accumulable — this is the TSQR (Tall Skinny QR) algorithm. Critically, R^T R = X^T X = C, the COPA centered cross-product matrix. So TSQR → eigendecompose R is algebraically equivalent to COPA → eigendecompose C. They are the same algorithm in disguise: both accumulate the Gram matrix; TSQR does it via R-factor stacking, COPA via the Welford/CGLeV merge.

The Householder obstruction is specifically about the LEFT singular vectors (U): recovering U requires the Q factors, which require non-associative composition. The RIGHT singular vectors (V) and singular values (Σ) pass through the Gram matrix and are fully accumulable.

**Consequence for tambear.** Any streaming SVD expressed as `accumulate(grouping, expr, op)` can produce V and Σ (via COPA or TSQR) but cannot produce U without a second data pass. This is a structural limitation of the semigroup framework, not an implementation gap.

### Open Direction: Large Square Matrices

For square matrices too large to fit in memory (n ≈ d ≈ 10⁴), neither COPA nor one-sided Jacobi fits. A randomized approach (random projection to reduce n before applying SVD) is the practical alternative:

```
Project: Y ← X · Ω    where Ω ∈ ℝᵈˣˡ is a random Gaussian matrix, l ≪ d
QR: Y = QR             Q ∈ ℝⁿˣˡ
Form: B ← Qᵀ X         B ∈ ℝˡˣᵈ
SVD: B = Ũ Σ Vᵀ
Approximate: U ≈ Q Ũ
```

This is the Halko-Martinsson-Tropp randomized SVD — it requires two passes over the data and l ≈ k + 10 random projections. It sits at the boundary between Kingdom A (the random projection and QR are deterministic) and Kingdom C (the approximation quality depends on the spectral gap, and power iteration improves it at the cost of additional passes).

## Connection to the Linearity Boundary Theorem

The COPA-SVD algorithm crystallizes the connection between the MSR principle and the linearity boundary.

### PCA as a Quadratic Objective

PCA minimizes reconstruction error: argmin_V ‖X̃ − X̃ VVᵀ‖²_F. This is a QUADRATIC objective in V (the Procrustes problem). Its solution is the top-k eigenvectors of X̃ᵀX̃ — a deterministic, closed-form function of the data.

The Hessian ∇²f is CONSTANT (does not depend on V for the equivalent formulation as maximizing projected variance: max tr(VᵀΣV) s.t. VᵀV = I). This is the defining characteristic of Kingdom A: constant Hessian → linear gradient → closed-form solution → one pass suffices.

### The MSR Exists Because the Objective is Quadratic

The 3-component MSR (n, μ, C) exists precisely because the PCA objective is quadratic. A quadratic objective f(V) = ½ Vᵀ A V − bᵀV has:

- Gradient ∇f = AV − b, which is LINEAR in V
- Solution V* = A⁻¹b, which is a DETERMINISTIC FUNCTION of A and b
- A and b are ACCUMULABLE from the data in one pass

For PCA: A = X̃ᵀX̃ = C (the COPA spread component), and the solution is the eigenvectors of C. The MSR (n, μ, C) contains exactly the information needed to state and solve the linear system — nothing more, nothing less.

### Nonlinear Objectives Have No Finite MSR

Contrast with t-SNE (also in dim_reduction.rs, Kingdom C). The t-SNE objective:

```
KL(P ‖ Q) = Σᵢⱼ pᵢⱼ log(pᵢⱼ / qᵢⱼ)
```

where Q depends on the EMBEDDING coordinates Y:

```
qᵢⱼ = (1 + ‖yᵢ − yⱼ‖²)⁻¹ / Σₖ≠ₗ (1 + ‖yₖ − yₗ‖²)⁻¹
```

The gradient ∇_{yᵢ} KL depends on ALL embedding coordinates {yⱼ}, which ARE the solution. The solution participates in its own definition — self-reference. No finite set of accumulable statistics suffices, because the gradient cannot be evaluated without the current estimate of the answer.

This is the linearity boundary theorem in action:

```
Quadratic objective → constant Hessian → linear gradient → closed-form MSR → Kingdom A
Nonlinear objective → solution-dependent Hessian → self-referential gradient → no MSR → Kingdom C
```

The EXISTENCE of a finite MSR is equivalent to the LINEARITY of the gradient, which is equivalent to the CONSTANCY of the Hessian. These are three views of the same mathematical property — the property that separates Kingdom A from Kingdom C.

### The Three-Layer Taxonomy

The full computation graph from data to output crosses three independent complexity layers, each governed by a different algebraic theory:

**Layer 1 — Semigroup (accumulation, sub-Fock).** Can the data be combined into a finite state via associative, commutative merge? This is the semigroup level. ALL moment accumulators — scalar M₂, COPA matrix C, co-skewness tensor M₃, co-kurtosis M₄ — merge commutatively regardless of rank. Accumulation is ALWAYS Kingdom A.

*Remark (Continuous semigroup families).* The Layer 1 semigroup is not uniquely determined by the problem — continuous families of valid semigroups may exist, parametrized by auxiliary quantities. For example, the logsumexp family (ℝ, ⊕_T) where x ⊕_T y = T·log(eˣ/ᵀ + eʸ/ᵀ) defines a commutative, associative operation for every T ∈ (0, ∞], interpolating between summation (T → ∞) and max (T → 0). All members of this family are valid Kingdom A accumulators: each is commutative and one-pass. The Kingdom classification — and therefore the Fock boundary — is invariant under semigroup choice within this family, because it depends on the Layer 2 extraction degree, not the Layer 1 accumulation type. Temperature T selects the oracle; the Kingdom is determined by the degree of what is extracted. This orthogonality between semigroup choice (Layer 1) and extraction complexity (Layer 2) is a non-obvious structural property of the three-layer taxonomy.

*Remark (Integer oracle limitation).* The oracle vocabulary as defined in this framework is restricted to real-valued numerical operations (addition, dot products, affine recurrences, Welford merge, etc.). Number-theoretic operations such as modular arithmetic (`mod`) are not in this vocabulary, despite appearing in algorithms — e.g., the Euclidean gcd algorithm — that are structurally analogous to real-valued counterparts such as Wynn's ε series accelerator. Both are KC→KB compositions: an iterative reduction phase (σ=1, τ=0) that generates a sequence of quotients, followed by a Kingdom B scan over the 2×2 matrix family `[[aₙ, 1], [1, 0]]` to evaluate the result. Both compute the best rational approximation in their respective algebraic domains (integer fractions for gcd; Padé approximants for Wynn ε). The present taxonomy covers the real-valued case. Extending it to integer oracles (over ℤ or ℤ_p) would unify these as the same Kingdom BC pattern operating over different number fields — a natural direction for future work.

**Layer 2 — Algebraic (extraction degree, super-Fock).** Is the extraction from MSR to output a closed-form function or an iterative search? This is determined by the objective degree:

| Objective degree | Hessian | Extraction | Kingdom |
|---|---|---|---|
| ≤ 2 (quadratic) | Constant (independent of solution) | Closed-form | A |
| ≥ 3 (cubic+) | Solution-dependent | Self-referential iteration | C |

The boundary is between degree 2 and degree 3. Quadratic objectives are the LAST with constant Hessians.

**Layer 3 — Galois (radical solvability, further super-Fock).** Even within Kingdom A extraction (unique deterministic answer), the COMPUTATIONAL METHOD may require iteration due to the algebraic structure of the characteristic polynomial:

| Dimension d | Galois group | Radical formula? | Extraction method |
|---|---|---|---|
| d ≤ 4 | Solvable | Yes (quadratic/Cardano/Ferrari) | Finite arithmetic |
| d ≥ 5 | S_d (non-solvable) | No (Abel-Ruffini, 1824) | Iterative (Jacobi/QR) |

This Galois-forced iteration is fundamentally different from Kingdom C iteration:

| | Kingdom C iteration | Galois-forced iteration |
|---|---|---|
| **Location** | Sub-Fock (data-touching) | Super-Fock (MSR-touching) |
| **Cost per step** | O(n) or O(n·d) | O(d³) |
| **Convergence** | May not converge; initialization-dependent | Always converges to unique answer |
| **Cause** | Self-referential gradient | Non-solvable Galois group |
| **Example** | t-SNE, IRLS on non-convex | Jacobi eigendecomposition |

For n >> d, the Galois-forced iteration cost (O(d³)) is negligible compared to the accumulation cost (O(n·d²)). For n ≈ d, the costs are comparable — this is precisely the crossover point where COPA should yield to direct SVD (Algorithm 1, Decision Criterion).

**Remark (Galois iteration ≠ Kingdom C iteration).** The distinction between these two iteration types is categorical, not merely quantitative. Galois-forced iteration operates on the d×d MSR (super-Fock), converges to a UNIQUE answer regardless of initialization, and its cost is independent of n. Kingdom C iteration operates on n-sized data structures (sub-Fock or straddling the Fock boundary), may converge to DIFFERENT fixed points depending on initialization, and its cost scales with n per iteration. A problem with Galois-forced extraction is Kingdom A — the problem has a unique closed-form answer; only the implementation iterates. A problem with Kingdom C iteration is genuinely iterative — the problem itself requires search. The three-layer taxonomy (semigroup/algebraic/Galois) captures this as three independent classification questions, not a linear progression.

### Theorem 7: The COPA Boundary

**Theorem 7 (COPA Boundary).** Let D be a multivariate output family computed from data X ∈ ℝⁿˣᵈ via an objective f.

(a) **MSR rank = objective degree.** The MSR rank of D equals the polynomial degree of the sufficient statistic for f. A degree-k objective requires a rank-k symmetric tensor MSR.

(b) **Kingdom A iff degree ≤ 2.** D is Kingdom A (one-pass accumulation, closed-form extraction from MSR) if and only if f is quadratic (degree ≤ 2). Quadratic objectives have constant Hessians; cubic and higher have solution-dependent Hessians.

(c) **COPA is the algebraic boundary.** The COPA state (n, μ, C) — rank-2 MSR with (p² + 3p + 2)/2 scalars — is the maximum-rank MSR whose consuming problem is Kingdom A. One step higher (rank-3 MSR for co-skewness), the objectives that USE co-skewness (ICA, blind source separation) have cubic+ objectives and are Kingdom C.

(d) **Galois obstruction for d ≥ 5.** COPA extraction (eigendecomposition of C/(n−1)) requires Galois-forced iteration for d ≥ 5 (Abel-Ruffini). This iteration is super-Fock (cost O(d³), independent of n) and converges to a unique answer regardless of initialization.

*Proof of (a).* A degree-k polynomial statistic is a function of {S₁, ..., Sₖ} (Theorem 1(a)). The centered versions {M₂, ..., Mₖ} are symmetric k-tensors in d dimensions. By Theorem 1(b), each Mⱼ is algebraically independent of the lower-order moments. Therefore the MSR must include all k tensor ranks. ∎

*Proof of (b).* A degree-2 objective f(x) = ½xᵀAx − bᵀx has Hessian ∇²f = A, which is constant. The optimality condition ∇f = Ax − b = 0 is a LINEAR system with unique solution x* = A⁻¹b — a deterministic function of A and b. Both are accumulable (A = XᵀX is a COPA merge, b = Xᵀy is a dot-product accumulate). For degree 3+: ∇²f is degree-(k-2) in the solution, so the gradient ∇f is nonlinear in the solution. The optimality condition is a nonlinear system requiring iterative solution. ∎

*Proof of (c).* By (a), co-skewness is rank-3 MSR. By (b), rank-2 is the last Kingdom A extraction. Co-skewness accumulation is still Kingdom A (Layer 1: commutative semigroup merge). But the problems that CONSUME co-skewness (e.g., ICA maximizes kurtosis: max E[(wᵀx)⁴] − 3, a degree-4 objective in w) have degree ≥ 3 objectives, making extraction Kingdom C. COPA is the boundary: highest rank where both accumulation AND extraction are Kingdom A. ∎

*Proof of (d).* The eigenvalues of the symmetric matrix Σ = C/(n−1) are the roots of det(Σ − λI) = 0, a degree-d polynomial. For d ≥ 5, the generic Galois group is S_d (non-solvable), so no radical formula exists (Abel-Ruffini, 1824). Iterative algorithms (Jacobi, QR) converge to these eigenvalues in O(d³) operations. Two distinct theorems guarantee this: (i) the **spectral theorem** guarantees that the eigenvalues exist, are real, and that eigenvectors form an orthonormal basis (because Σ is symmetric); (ii) the **Jacobi convergence theorem** (Jacobi, 1846) guarantees that Jacobi rotations converge — the off-diagonal Frobenius norm decreases monotonically to zero. These are independent guarantees: the spectral theorem is about the PROBLEM (eigenvalues exist), Jacobi convergence is about the ALGORITHM (iterates converge to them). This cost is independent of n because the iteration operates on the d×d MSR, not the n×d data. ∎

**Remark (Information-Theoretic Interpretation of the COPA Boundary).** The COPA state (n, μ, C) coincides with the sufficient statistics of the d-dimensional Gaussian distribution N(μ, C/(n−1)). This is not accidental: the Gaussian maximizes entropy H(X) subject to fixed mean and covariance (Jaynes, 1957). Kingdom A extraction (degree ≤ 2) corresponds precisely to the maximum-entropy component of the data's statistical structure — the part explainable by the best-fit Gaussian model. Non-Gaussian structure (skewness, kurtosis, multimodality) requires degree ≥ 3 objectives, placing their extraction in Kingdom C. The Fock boundary thus separates maximum-entropy analysis (linear, one-pass, Gaussian) from sub-maximum-entropy analysis (nonlinear, iterative, non-Gaussian). This separation is reinforced by the central limit theorem: accumulation (the defining operation of Kingdom A) drives toward Gaussianity, making Kingdom A naturally aligned with the statistical structure that summation produces. Kingdom C methods, which seek non-Gaussian structure, work against the CLT — they extract information that accumulation destroys.

**Remark (EM convergence as quotient severity).** By Dempster, Laird & Rubin (1977), the convergence rate of the EM algorithm equals the spectral norm of the missing information matrix divided by the complete information matrix: ρ = λ_max(I_missing / I_complete). This is precisely the severity of the quotient from complete to incomplete data sufficient statistics. A more severe quotient — a larger gap between what the complete data MSR would provide and what the observed data MSR contains — produces slower EM convergence. The COPA boundary theorem specifies the maximum-rank MSR available in one pass; using a lower-rank MSR than necessary increases the quotient severity and slows any iterative estimation that follows.

**Remark (Galois-forced iteration is NOT Kingdom C).** The iteration in Theorem 7(d) must not be confused with Kingdom C iteration. They differ in every structural property:

| Property | Galois-forced (Thm 7(d)) | Kingdom C |
|---|---|---|
| **Operates on** | d×d MSR (super-Fock) | n-sized data structures (sub-Fock) |
| **Cost** | O(d³) total | O(n) per iteration |
| **Convergence** | Guaranteed unique answer (spectral theorem: eigenvalues exist and are real; Jacobi convergence: iterates converge to them) | May not converge; initialization-dependent for non-convex objectives |
| **Solution uniqueness (τ)** | τ = 0 always | τ = 0 (convex) or τ = 1 (non-convex) |
| **Cause** | Abel-Ruffini theorem (Galois group S_d non-solvable for d ≥ 5) | Self-referential gradient (solution participates in its own computation) |
| **Problem classification** | Kingdom A (constant Hessian, degree-2 objective) | Kingdom C (solution-dependent Hessian, degree ≥ 3 objective) |

A Kingdom A problem with Galois-forced extraction remains Kingdom A — the problem has a unique closed-form answer determined entirely by the data. Only the extraction algorithm iterates, on the compact MSR, not the data. This distinction is categorical: the three-layer taxonomy (semigroup/algebraic/Galois) classifies the PROBLEM (intrinsic), while the algorithmic classification (ρ,σ,τ) describes the ALGORITHM (contingent — the same d=3 eigendecomposition can be solved without iteration via Cardano, but d ≥ 5 forces iterative extraction by a theorem of algebra).

**Corollary.** The accumulation/extraction cost crossover occurs at n ≈ d: when n·d² (accumulation) ≈ d³ (Galois extraction), i.e., n ≈ d. For n >> d, accumulation dominates and COPA is optimal. For n ≈ d, extraction dominates and direct SVD (avoiding the κ² squaring of COPA's Gram matrix path) is preferred.

**Connection to Fock boundary density:**

- Kingdom A (constant Hessian): one Fock collapse, at the end. Maximum laziness. PCA collapses once — all data summarized in COPA state, then extracted.
- Kingdom B (constant but non-commutative Hessian): one Fock collapse per data point. O(n) materializations.
- Kingdom C (solution-dependent Hessian): one Fock collapse per iteration. O(n · iter) materializations. t-SNE must materialize the embedding at every iteration.

The Galois-forced iteration within Kingdom A is a FURTHER collapse — O(d³/d²) ≈ O(d) additional materializations within the super-Fock extraction. But these are d-sized, not n-sized. The Fock boundary separates the two iteration scales.

The 3-component MSR (n, μ, C) is the Kingdom A manifestation: the point at which superposition collapses exactly once to yield a finite, mergeable, sufficient representation — the algebraic boundary of one-pass multivariate analysis.

### Theorem 8: Fock Boundary Characterization

**Theorem 8 (Fock Boundary Characterization).** For a statistical estimation problem with polynomial objective f(θ; S) on parameter space Θ ⊆ ℝᵈ, where S denotes the accumulated sufficient statistics, the following are equivalent:

(a) **Kingdom A.** The optimal θ* is a deterministic, closed-form function of a finite accumulable MSR.

(b) **Quadratic objective.** The objective f has polynomial degree ≤ 2 in θ.

(c) **Constant Hessian.** The Hessian ∇²_θ f is independent of θ (the gradient is linear in the parameters).

(d) **Flat optimization landscape.** The Riemannian manifold (Θ, g) with metric tensor g = ∇²_θ f has zero curvature — all level sets of f are concentric ellipsoids, and geodesics are straight lines.

Moreover, when the MSR has rank 2, its components (n, μ, C) coincide with the sufficient statistics of the Gaussian distribution N(μ, C/(n−1)) — the unique maximum-entropy distribution for fixed mean and covariance (Jaynes, 1957).

*Proof.* (b) ↔ (c): A polynomial of degree ≤ 2 has constant second derivatives by direct computation. Conversely, if all second derivatives are constant, the Taylor expansion terminates at degree 2. (b) ↔ (d): A constant Hessian, interpreted as a Riemannian metric tensor, has all Christoffel symbols ∂gᵢⱼ/∂θₖ = 0, hence zero Riemann curvature. The level sets {θ : f(θ) = c} of a quadratic form are concentric ellipsoids (or hyperplanes in degenerate cases). Conversely, non-zero curvature implies the Hessian varies, so deg(f) ≥ 3. (b) → (a): This is Theorem 7(b). The optimality condition ∇f = Aθ − b = 0 is linear, with solution θ* = A⁻¹b, a deterministic function of A and b, which are accumulable. (a) → (b) [for polynomial objectives]: If deg(f) ≥ 3, then ∇²f depends on θ, so ∇f is nonlinear in θ. The optimality condition is a nonlinear system whose solution is self-referential: evaluating the gradient requires the current parameter estimate. This is the defining characteristic of Kingdom C (Theorem 7(b)). For the Gaussian characterization: the rank-2 MSR (n, μ, C) has components that are exactly {count, mean, centered outer product sum}. The Gaussian N(μ, Σ) with Σ = C/(n−1) has log-likelihood ℓ(μ, Σ) whose sufficient statistics are precisely {n, Σxᵢ, Σxᵢxᵢᵀ} — equivalent to (n, μ, C) via invertible transformation. By the maximum-entropy theorem, the Gaussian is the unique distribution maximizing entropy subject to these second-moment constraints. ∎

**Remark (Scope: polynomial objectives).** The restriction to polynomial objectives in Theorem 8 is essential. Exponential families provide an alternative route to Kingdom A that bypasses the quadratic structure entirely. For any exponential family p(x|η) = h(x) exp(ηᵀT(x) − A(η)), the MLE equation reduces to ∇A(η) = T̄(x) — matching the expected sufficient statistic to the observed average. This yields closed-form MLE (Kingdom A extraction) even when the log-likelihood is transcendental and the Fisher-Rao manifold is curved. The Bernoulli (p̂ = x̄), Poisson (λ̂ = x̄), and exponential (β̂ = x̄) all exemplify this: Kingdom A by the exponential family theorem, not by quadratic structure. The Fock boundary characterization (Theorem 8) applies to the polynomial world — PCA, regression, ANOVA, all quadratic optimization problems. The exponential family world has its own Kingdom A mechanism, governed by the convexity of the log-partition function A(η) rather than by the degree of the objective.

**Remark (The CLT as structural explanation).** Why is the Fock boundary at degree 2 and not elsewhere? The central limit theorem provides a structural answer: accumulation (summation of independent contributions) drives distributions toward the Gaussian, which is the maximum-entropy distribution for fixed second moments. Kingdom A IS accumulation (commutative, associative merge). The CLT says the output of accumulation is Gaussian-like. Therefore Kingdom A naturally produces and extracts the statistical structure (mean, covariance) that accumulation generates. Kingdom C methods — which seek non-Gaussian structure (skewness, kurtosis, multimodality) — work against the CLT: they extract information that summation destroys. This is not a metaphor but a consequence of the maximum-entropy principle: the Gaussian is the fixed point of additive processes under second-moment constraints, and the Fock boundary is where the accumulated statistics exhaust the Gaussian model.

### Two Phase Boundaries of the Algebraic Hierarchy

**Remark (Two Phase Boundaries).** The algebraic hierarchy of computational problems has two sharp, intrinsic phase boundaries:

**1. The Fock boundary (degree 2 → 3).** Separates Kingdom A from Kingdom C. Below: quadratic objectives with constant Hessians, closed-form MSR, one-pass accumulation. Above: cubic+ objectives with solution-dependent Hessians, self-referential gradients, iterative search. The boundary is sharp — there is no "degree 2.5" objective. The COPA state (n, μ, C) sits exactly on this boundary: the maximum-rank MSR whose extraction remains Kingdom A (Theorem 7(c)).

**2. The Banach boundary (contractive → non-contractive).** Separates τ = 0 from τ = 1 within Kingdom C. Below: problems where the iteration operator is a contraction (ρ < 1), so the Banach fixed-point theorem guarantees a unique attractor — iteration converges to the same answer regardless of initialization (e.g., IRLS on convex GLMs). Above: problems where the contraction condition fails and multiple attractors exist — different initializations may converge to different fixed points (e.g., k-means, ICA, non-convex neural network training). Convexity of the objective is *sufficient* for τ = 0 (convexity implies a unique global minimum, hence a unique attractor), but *not necessary*: a non-convex objective may still have τ = 0 if all trajectories of the iteration operator converge to the same basin. The boundary is at ρ_contraction = 1, not at the loss of convexity per se.

These two boundaries are independent and intrinsic to the problem:

| | Below boundary | Above boundary | What changes |
|---|---|---|---|
| **Fock boundary** | Degree ≤ 2 (Kingdom A) | Degree ≥ 3 (Kingdom C) | Hessian becomes solution-dependent |
| **Banach boundary** | Contractive (τ = 0, typically convex) | Non-contractive (τ = 1) | Solution uniqueness is lost |

The Fock boundary determines WHETHER iteration is needed (constant vs. solution-dependent Hessian). The Banach boundary determines WHETHER iteration has a unique answer (convex vs. non-convex landscape). Together they partition the computational landscape into three regimes:

```
Kingdom A (degree ≤ 2):     No iteration. Unique answer. One-pass MSR.
Kingdom C, τ=0 (contractive): Iteration required. Unique attractor. Converges from any start.
Kingdom C, τ=1 (non-contractive): Iteration required. Multiple attractors. Initialization-dependent.
```

The naming reflects the mathematics: the Fock boundary is where the Fock space structure (superposition → collapse) determines accumulation feasibility. The Banach boundary is where the Banach fixed-point theorem determines convergence uniqueness — below it (ρ < 1), the contraction mapping theorem guarantees a unique fixed point; above it (ρ ≥ 1), the theorem's conditions fail and multiple attractors may emerge. Convexity implies ρ < 1 (and is the most common reason for τ = 0 in practice), but the Banach boundary is properly at ρ = 1, not at the convexity/non-convexity transition.

Note that Kingdom B (sequential, non-commutative) is orthogonal to both boundaries — it concerns the commutativity of the semigroup, not the degree of the objective or the convexity of the landscape. The full classification requires all three axes: commutativity (A vs. B), objective degree (Fock boundary), and convexity (Banach boundary).
